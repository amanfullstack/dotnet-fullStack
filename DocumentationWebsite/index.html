🔥 Top 10 Quick Questions & Answers

1 What is async/await and why is it important? +

Answer: Async/await allows non-blocking code execution. While waiting for I/O (database, network), the thread can handle other requests. Essential for scalability!

public async Task<List<Product>> GetProductsAsync()
{
    // Thread released here, can handle other requests
    return await _service.GetAllProductsAsync();
}
2 Explain N-Tier Architecture +

Answer: N-Tier separates concerns: Presentation (UI/Controllers) → Business Logic (Services) → Data Access (DbContext) → Database. Each layer has single responsibility.

3 When would you use EF Core vs ADO.NET? +

Answer: EF Core for standard CRUD (rapid development, LINQ). ADO.NET for complex queries, stored procedures, maximum performance. GraphQL for flexible client queries.

4 What are the 5 caching strategies? +

Answer: (1) In-Memory Cache, (2) HTTP Cache Headers, (3) Response Caching Middleware, (4) Redis (distributed), (5) EF Query Compilation.

5 What is GraphQL and its advantages? +

Answer: GraphQL is a query language. Advantages: no over/under-fetching, self-documenting schema, single endpoint, better for mobile (less bandwidth), perfect for multiple clients with varying needs.

6 Explain Dependency Injection +

Answer: DI is a pattern where dependencies are injected rather than created. Benefits: testability (use mocks), loose coupling, flexibility (swap implementations).

7 What is middleware and how does it work? +

Answer: Middleware is a component in the request pipeline. Each middleware can inspect/modify request, call next middleware, or short-circuit pipeline. Example: error handling, logging, authentication.

8 How would you optimize a slow API? +

Answer: (1) Profile to find bottleneck, (2) Add caching (in-memory or Redis), (3) Optimize queries (eager loading, indexes), (4) Use async/await, (5) Consider ADO.NET or pagination.

9 What is soft delete and why use it? +

Answer: Soft delete sets IsActive=false instead of hard delete. Benefits: preserve audit history, recover data, maintain referential integrity, comply with data retention policies.

10 What is the N+1 query problem? +

Answer: 1 query gets records, then N more queries get related data = 1+N queries. Solution: eager loading (.Include()), batch loaders, or single optimized query. Critical for performance!
