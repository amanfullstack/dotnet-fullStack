🅰️ Angular Setup Guide

Complete step-by-step guide to set up and run the ProductUI-Angular project locally.

Prerequisites

⚠️

Before starting, ensure you have:

  • ✅ Node.js 18+ and npm 9+ installed
  • ✅ Angular CLI installed globally: npm install -g @angular/cli
  • ✅ ProductCatalogAPI running on localhost:5000
  • ✅ Git (for cloning the repository)

1. Check Prerequisites

# Check Node.js version (should be 18+)
node --version

# Check npm version (should be 9+)
npm --version

# Check Angular CLI
ng version

2. Clone or Navigate to Project

# Using Git
git clone https://github.com/[USER]/dotnet-fullStack.git
cd dotnet-fullStack/ProductUI-Angular-TypeScript

# Or if already cloned
cd ProductUI-Angular-TypeScript

3. Verify API is Running

The Angular app depends on the ProductCatalogAPI. Make sure it's running:

cd ProductCatalogAPI-WebAPI-EFCore
dotnet run

# Should show: "Now listening on: http://localhost:5000"

4. Install Dependencies

# Install npm packages (takes 2-3 minutes)
npm install

# Verify installation completed successfully
npm --version
ls node_modules | head

5. Configure Environment

Verify API endpoint in src/environments/environment.ts:

export const environment = {
  production: false,
  apiUrl: 'http://localhost:5000/api'
};

6. Start Development Server

# Start Angular dev server
ng serve

# Or with automatic browser open
ng serve --open

# Server starts on http://localhost:4200

7. Access the Application

  • Home Page: http://localhost:4200
  • Products List: http://localhost:4200/
  • Create Product: http://localhost:4200/create
  • Edit Product: http://localhost:4200/edit/:id
  • Product Detail: http://localhost:4200/:id

Features

✓

Once running, you can:

  • 📋 View all products in a responsive table
  • 🔍 Search products by name or category
  • ➕ Create new products via reactive forms
  • ✏️ Edit existing product details
  • 👁️ View individual product details
  • 🗑️ Delete products with confirmation
  • 🌙 Toggle dark/light theme
  • 📱 Full responsive design for mobile/tablet
  • ⚡ Real-time validation with Reactive Forms
  • 🎯 RxJS observables for async operations

Troubleshooting

Issue: Cannot connect to ProductCatalogAPI

Solution: Ensure the API is running on localhost:5000

# In another terminal window, run:
cd ProductCatalogAPI-WebAPI-EFCore
dotnet run

Issue: CORS Error from API

Solution: The API should already have CORS configured for localhost:4200, but if not:

# Check ProductCatalogAPI Program.cs for CORS setup
# Ensure it includes: "http://localhost:4200"

Issue: Port 4200 already in use

Solution: Use a different port

# Run on custom port
ng serve --port 4300

Development Workflow

# Dev server with hot reload
ng serve

# Build for production
ng build --configuration production

# Run unit tests
ng test

# Run end-to-end tests
ng e2e

  • Netlify: ng build → drag dist/ folder
  • Vercel: Connect GitHub repo, auto-deploys
  • GitHub Pages: Configure baseHref in angular.json
