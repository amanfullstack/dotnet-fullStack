🗺️ React Router

Master React Router for client-side navigation and routing.

What Is React Router?

React Router v6 is the standard library for handling navigation in React single-page applications. It enables bookmarkable URLs, browser back/forward buttons, and multi-page app experiences without full page reloads.

Think of React Router as the navigation system for your React app - managing which component shows for which URL.

Why Router Matters

  • Bookmarkable URLs: Users can save and share app links
  • Browser Navigation: Back/forward buttons work naturally
  • Multi-Page Feel: Create apps with distinct pages/sections
  • Code Splitting: Load page modules on demand
  • Protected Routes: Restrict access to certain pages

Key Concepts

🌐 BrowserRouter

Wraps your app to enable routing:

<BrowserRouter>
  <App />
</BrowserRouter>

🛣️ Routes & Route

Define which component shows for each URL:

<Routes>
  <Route path="/" element={<Home />} />
  <Route path="/products" element={<Products />} />
  <Route path="/products/:id" element={<Detail />} />
</Routes>

🔗 Link & Navigate

  • <Link to="/path"> - Navigation link
  • useNavigate() - Programmatic navigation
  • <Navigate to="/path" /> - Redirect

📍 URL Params & Query

Get data from URL:

const { id } = useParams(); // /products/123
const [searchParams] = useSearchParams(); // /products?sort=price

Code Examples

Example 1: Basic Routing Setup

import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

// App setup
function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/products">Products</Link>
        <Link to="/about">About</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/products" element={<Products />} />
        <Route path="/about" element={<About />} />
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}

// Page components
function Home() { return <h1>Welcome Home!</h1>; }
function Products() { return <h1>Our Products</h1>; }
function NotFound() { return <h1>404 - Not Found</h1>; }

Example 2: Dynamic Routes with Params

import { useParams } from 'react-router-dom';

// Route definition
<Route path="/products/:id" element={<ProductDetail />} />

// Component using params
function ProductDetail() {
  const { id } = useParams();

  const [product, setProduct] = useState(null);

  useEffect(() => {
    // Fetch product by ID
    fetch(`http://localhost:5000/api/products/${id}`)
      .then(r => r.json())
      .then(data => setProduct(data));
  }, [id]);

  if (!product) return <div>Loading...</div>;

  return (
    <div>
      <h1>{product.name}</h1>
      <p>Price: ${product.price}</p>
      <!-- Navigation to other product -->
      <Link to={`/products/${product.id + 1}`}>
        Next Product
      </Link>
    </div>
  );
}

Example 3: Nested Routes & Layouts

<Routes>
  <Route path="/" element={<Layout />}>
    <Route index element={<Home />} />
    <Route path="products" element={<Products />} />
    <Route path="products/:id" element={<Detail />} />
  </Route>
</Routes>

// Layout wraps all pages
function Layout() {
  return (
    <div>
      <header>Header</header>
      <Outlet /> {/* Child routes render here */}
      <footer>Footer</footer>
    </div>
  );
}

Example 4: Protected Routes with Auth

function ProtectedRoute({ children }) {
  const { user } = useContext(AuthContext);

  if (!user) {
    return <Navigate to="/login" />;
  }

  return children;
}

// Usage
<Route
  path="/dashboard"
  element={
    <ProtectedRoute>
      <Dashboard />
    </ProtectedRoute>
  }
/>

✅ Best Practices

1. Use Link for Internal Navigation

Why: No page reload, preserves app state

2. Handle 404 with Wildcard Route

Why: Shows friendly error instead of blank

<Route path="*" element={<NotFound />} />

3. Use useNavigate for Programmatic Nav

Why: Navigate after form submission, API call, etc.

const navigate = useNavigate();
navigate('/success');
navigate(-1); // Go back

4. Lazy Load Routes

Why: Reduces initial bundle size

const Admin = lazy(() => import('./Admin'));
<Route path="/admin" element={<Suspense><Admin /></Suspense>} />

❌ Common Mistakes

1. Forgetting BrowserRouter Wrapper

Routes won't work without BrowserRouter at top level

2. Using <a> Instead of <Link>

// ❌ Causes full page reload
<a href="/products">Products</a>

// ✅ Client-side navigation
<Link to="/products">Products</Link>

3. Not Using activeClassName

Highlight current page with NavLink instead of Link

import { NavLink } from 'react-router-dom';
<NavLink to="/products" className={({ isActive }) => isActive ? 'active' : ''}>
  Products
</NavLink>

🎓 Interview Q&A

Q1: What's the difference between Link and <a> tags?

Answer: Link does client-side navigation (no page reload). <a> causes full page reload. Use Link for internal navigation.

Q2: How do you protect routes from unauthorized access?

Answer: Create a ProtectedRoute component that checks if user is logged in, redirects to login if not

Q3: What is useParams and how do you use it?

Answer: useParams gets URL parameters from the route (e.g., /products/:id). Returns object with parameter values.
