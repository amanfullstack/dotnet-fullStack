🌍 Context API

Manage global state with React Context API and providers.

What Is Context API?

Context API provides a way to pass data through the component tree without manually passing props at every level. It's perfect for sharing data like user authentication, theme preferences, or language settings across your entire app.

Instead of prop drilling (passing props through many layers), use Context to make data globally available.

Why Context API Matters

  • Avoid Prop Drilling: No need to pass props through every component level
  • Global State: Share data across entire app easily
  • Theme Management: Switch themes without touching props
  • Authentication: Access user data from any component
  • Cleaner Code: Components don't pass irrelevant props

Key Concepts

📦 createContext

Creates a context object to hold data:

const ThemeContext = createContext('light');

🔌 Provider

Wraps components and provides context value:

<ThemeContext.Provider value={{ theme, setTheme }}>
  <App />
</ThemeContext.Provider>

🎯 useContext Hook

Consume context value in any nested component:

const { theme } = useContext(ThemeContext);

⚠️ When NOT to Use

  • Frequently changing data (causes re-renders)
  • Complex state management (use Redux instead)
  • Simple prop passing between 1-2 levels

Code Examples

Example 1: Theme Context

import { createContext, useState, useContext } from 'react';

// Create context
const ThemeContext = createContext();

// Provider component
export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// Custom hook
export function useTheme() {
  return useContext(ThemeContext);
}

// App setup
function App() {
  return (
    <ThemeProvider>
      <Header />
    </ThemeProvider>
  );
}

// Any component can use theme
function Button() {
  const { theme, toggleTheme } = useTheme();
  return (
    <button
      onClick={toggleTheme}
      style={{ background: theme === 'light' ? '#fff' : '#333' }}
    >
      Toggle Theme
    </button>
  );
}

Example 2: Authentication Context

const AuthContext = createContext();

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Check if logged in
    const checkAuth = async () => {
      try {
        const response = await fetch('http://localhost:5000/api/me');
        if (response.ok) {
          const data = await response.json();
          setUser(data);
        }
      } finally {
        setLoading(false);
      }
    };
    checkAuth();
  }, []);

  return (
    <AuthContext.Provider value={{ user, loading }}>
      {children}
    </AuthContext.Provider>
  );
}

// Protected component
function Dashboard() {
  const { user, loading } = useContext(AuthContext);
  if (loading) return <div>Loading...</div>;
  if (!user) return <Navigate to="/login" />;
  return <h1>Welcome, {user.name}!</h1>;
}

Example 3: Notification Context

const NotificationContext = createContext();

export function NotificationProvider({ children }) {
  const [notifications, setNotifications] = useState([]);

  const addNotification = (message, type = 'info') => {
    const id = Date.now();
    setNotifications(prev => [...prev, { id, message, type }]);
    setTimeout(() => {
      setNotifications(prev => prev.filter(n => n.id !== id));
    }, 3000);
  };

  return (
    <NotificationContext.Provider value={{ addNotification }}>
      {children}
      <NotificationList notifications={notifications} />
    </NotificationContext.Provider>
  );
}

// Usage
function SaveForm() {
  const { addNotification } = useContext(NotificationContext);

  const handleSave = async () => {
    try {
      await fetch('/api/save', { method: 'POST' });
      addNotification('Saved!', 'success');
    } catch (err) {
      addNotification('Error: ' + err.message, 'error');
    }
  };

  return <button onClick={handleSave}>Save</button>;
}

✅ Best Practices

1. Always Create Custom Hook Wrapper

Why: Cleaner and enforces that providers exist

2. Split Large Contexts

Why: Context changes cause all consumers to re-render

Use separate contexts for theme, auth, and notifications

3. Memoize Context Value

Why: Prevents unnecessary re-renders

🎓 Interview Q&A

Q1: What's prop drilling and how does Context solve it?

Answer: Prop drilling means passing props through many intermediate components. Context makes data globally available, avoiding this.

Q2: When should you use Context vs Redux?

Answer: Context for simple state (theme, auth). Redux for complex apps with many state changes and debugging needs.

Q3: Why do all consumers re-render when context changes?

Answer: React can't know which consumers only need part of it. Solution: split into smaller contexts or memoize.
