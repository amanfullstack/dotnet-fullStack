🖥️ ASP.NET MVC Setup Guide

Complete step-by-step guide to set up and run the ProductCatalogMVC project locally.

Prerequisites

⚠️

Before starting, ensure you have:

  • ✅ .NET 8.0 SDK or later installed
  • ✅ Visual Studio 2022 or VS Code with C# extension
  • ✅ ProductCatalogAPI running on localhost:5000
  • ✅ Git (for cloning the repository)

1. Clone or Navigate to Project

# Using Git
git clone https://github.com/[USER]/dotnet-fullStack.git
cd dotnet-fullStack/ProductCatalogMVC-ASP.NET-MVC

# Or if already cloned
cd ProductCatalogMVC-ASP.NET-MVC

2. Verify API is Running

The MVC project depends on the ProductCatalogAPI. Make sure it's running:

cd ProductCatalogAPI-WebAPI-EFCore
dotnet run

# Should show: "Now listening on: http://localhost:5000"

3. Restore Dependencies

# Restore NuGet packages
dotnet restore

# Or in Visual Studio: Build > Rebuild Solution

4. Configure Application

Edit appsettings.json if needed:

{
  "Logging": {
    "LogLevel": {
      "Default": "Information"
    }
  },
  "ProductCatalogApi": {
    "BaseUrl": "http://localhost:5000"
  }
}

5. Run the Application

# Start the development server
dotnet run

# Or in Visual Studio: Press F5 or Debug > Start Debugging

# Application starts on:
# - HTTP: http://localhost:5000
# - HTTPS: https://localhost:5001

6. Access the Application

Open your browser to:

  • Home Page: https://localhost:5001 (or http://localhost:5000)
  • Products List: https://localhost:5001/Products
  • Create New Product: https://localhost:5001/Products/Create

Features

✓

Once running, you can:

  • 📋 View all products in a responsive table
  • 🔍 Search products by name or category
  • ➕ Create new products via form
  • ✏️ Edit existing product details
  • 👁️ View individual product details
  • 🗑️ Delete products with confirmation
  • 🌙 Toggle dark/light theme
  • 📱 Full responsive design for mobile/tablet

Troubleshooting

Issue: Cannot connect to ProductCatalogAPI

Solution: Ensure the API is running on localhost:5000

# In another terminal window, run:
cd ProductCatalogAPI-WebAPI-EFCore
dotnet run

Issue: Port already in use

Solution: Change the launch URL in launchSettings.json

"applicationUrl": "https://localhost:5002;http://localhost:5001"

Issue: SSL/TLS certificate error

Solution: Trust the development certificate

dotnet dev-certs https --trust

Development Workflow

# Watch for changes and auto-reload
dotnet watch run

# Build only
dotnet build

# Publish for production
dotnet publish -c Release

├── Views/
│   ├── Shared/
│   │   └── _Layout.cshtml        (Master layout)
│   └── Products/

│       ├── Edit.cshtml           (Edit product form)
│       ├── Delete.cshtml         (Confirmation)
│       └── Details.cshtml        (Product details)
├── Services/
│   ├── IProductService.cs        (Interface)

├── wwwroot/
│   ├── css/
│   │   ├── site.css              (Custom styles)
│   │   └── theme.css             (Dark/light theme)
│   └── js/
│       └── theme.js              (Theme toggle)
├── Program.cs                    (DI & middleware setup)
└── appsettings.json              (Configuration)
