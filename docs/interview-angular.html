    🅰️ Angular Fundamentals

    Full-featured TypeScript framework — components, services, DI, and RxJS for Product Catalog frontend

    Advanced Interview Prep 9 Core Concepts
    🅰️

    What is Angular?

    Angular is an opinionated, full-featured framework for building enterprise-scale web applications. Unlike React (a library), Angular forces best practices: Dependency Injection, TypeScript strict typing, modules, decorators, and RxJS for async data. The learning curve is steeper than React, but Angular scales to massive codebases with hundreds of developers. Every Angular app follows the same architecture — there is no "Angular way" and "another way." This rigidity is a feature, not a bug. If you are building Product Catalog as an SPA (Single Page App), Angular handles routing, forms, HTTP, state, testing — everything is built-in.

    Quick Reference: Angular Elements

    Angular Element Role TypeScript Pattern
    Component UI + logic for a screen or reusable widget @Component class with template, styles, selector
    Service Shared business logic, HTTP calls, data @Injectable class with methods; injected via constructor
    Module (NgModule) Container for components, services, directives @NgModule with declarations, imports, providers, exports
    Directive Extends HTML behavior; structural (*ngIf) or attribute ([ngClass]) @Directive with selector, modifies DOM or element
    RxJS Observable Stream of async events (HTTP, timers, user input) Observable<T>; subscribe or use async pipe
    Dependency Injection (DI) Framework injects dependencies into components/services constructor(private svc: ProductService) — DI provides svc
    Router Client-side navigation, component lazy-loading Routes array, router.navigate(), ActivatedRoute
    Form Handle user input with validation and binding Template-driven (2-way [(ngModel)]) or Reactive (FormBuilder)
    Lifecycle Hook Run code at specific component lifecycle stages ngOnInit, ngOnChanges, ngOnDestroy, etc.

    Angular Fundamentals Interview Questions

    Q1. What is Angular? How does it compare to React? ▼

    Angular is an opinionated, full-featured framework for building enterprise Single Page Applications (SPAs). It includes routing, forms, HTTP client, testing tools, and state management out of the box. Angular is written in TypeScript and enforces strict typing.

    React is a library focused only on the view layer (components). You must bring your own router, HTTP client, form library, and state management. This flexibility appeals to developers who like control; however, you must evaluate and integrate third-party libraries for every feature.

    AspectAngularReact
    TypeFull frameworkView library
    LanguageTypeScript (mandatory)JavaScript or TypeScript (optional)
    OpinionsHighly opinionated; one way to do thingsUnopinionated; many ways
    Learning curveSteep (decorators, DI, RxJS)Gentle (JSX, props, hooks)
    RoutingRouterModule (built-in)React Router (third-party)
    StateServices + RxJS ObservablesRedux, Zustand, Context (pick one)
    FormsReactiveFormsModule or template-drivenFormik, React Hook Form (third-party)
    Bundle size~1.2 MB gzipped (full framework)~42 KB gzipped (core only)
    Use caseLarge enterprise apps, 100+ devsStartups, fast iterations, flexibility

    Which for Product Catalog? If the frontend is complex (many pages, forms, real-time data, large team), Angular scales well. If you want quick iteration and small team, React is lighter and more flexible.

    Q2. What are Angular components, services, and modules? ▼

    Angular's N-tier architecture separates concerns into three core building blocks:

    Components are UI + logic. A component has a template (HTML), styles (CSS), and class (TypeScript). Example: ProductListComponent displays a table of products and handles click events.

    Services are shared business logic. They do NOT have UI. Services handle HTTP calls, data transformations, caching, API communication. All components that need products inject ProductService and call its methods.

    Modules (NgModule) are containers that group related components, services, and directives. The root module is AppModule. Large apps have many feature modules (ProductModule, CartModule, OrderModule) to organize code and enable lazy-loading.

    typescript — Component, Service, Module structure 
    // product.service.ts — PURE SERVICE, NO UI
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

export interface Product {
  id: number;
  name: string;
  price: decimal;
  category: string;
  stock: number;
}

@Injectable({
  providedIn: 'root' // globally available
})
export class ProductService {
  private apiUrl = 'http://localhost:5000/api/products';

  constructor(private http: HttpClient) { }

  // Observable = async stream. Caller subscribes.
  getAll(): Observable<Product[]> {
    return this.http.get<Product[]>(`${this.apiUrl}`);
  }

  getById(id: number): Observable<Product> {
    return this.http.get<Product>(`${this.apiUrl}/${id}`);
  }

  create(product: Product): Observable<Product> {
    return this.http.post<Product>(`${this.apiUrl}`, product);
  }
}

    Q3. What is Dependency Injection in Angular? ▼

    Dependency Injection (DI) is a design pattern where an object requests its dependencies from an external container instead of creating them itself. Angular's DI framework automatically wires up services and injects them into components and services.

    How it works:

    1. Mark a class as injectable with @Injectable() decorator.
    2. In a component, declare the dependency in the constructor with a type annotation.
    3. Angular DI container creates one instance (or reuses one based on scope) and passes it to the constructor.
    4. The component accesses the service via a private property.
    typescript — Dependency Injection example 
    // CacheService — a dependency
@Injectable({ providedIn: 'root' })
export class CacheService {
  private cache = new Map<string, any>();

  set(key: string, value: any): void {
    this.cache.set(key, value);
  }

  get(key: string): any | undefined {
    return this.cache.get(key);
  }
}

// ─────────────────────────────────────────────────────────

// ProductService — DEPENDS ON CacheService
@Injectable({ providedIn: 'root' })
export class ProductService {
  constructor(
    private http: HttpClient,
    private cache: CacheService  // ← DI injects CacheService
  ) { }

  getAll(): Observable<Product[]> {
    // Check cache first
    const cached = this.cache.get('products:all');
    if (cached) return of(cached); // of() wraps sync value in Observable

    // Cache miss — fetch from API
    return this.http.get<Product[]>('/api/products').pipe(
      tap((products) => this.cache.set('products:all', products))
    );
  }
}

// ─────────────────────────────────────────────────────────

    Benefits: Loose coupling (components do not know concrete implementations); testability (inject mock services in tests); flexibility (swap implementations by changing one line in the module).

    DI Scopes: providedIn: 'root' = singleton (one instance app-wide); providedIn: SomeModule = one per module; no provider = new instance each injection.

    Q4. What is data binding in Angular? ▼

    Data binding connects the component class (TypeScript) to the template (HTML). There are four types:

    1. Interpolation {{ }} — one-way: display component property in HTML.

    2. Property binding [prop] — one-way: set an HTML element property from component.

    3. Event binding (click) — one-way: respond to user events (click, input, change).

    4. Two-way binding [(ngModel)] — both directions: changes in HTML update component, and vice versa.

    typescript — Data binding in ProductListComponent 
    // product-list.component.ts
@Component({
  selector: 'app-product-list',
  template: `
    <div class="filter">
      <!-- Two-way binding: input updates searchText, searchText updates input -->
      <input [(ngModel)]="searchText" placeholder="Search products...">
      <button (click)="onSearch()">Search</button>
    </div>

    <table>
      <tr *ngFor="let product of filteredProducts">
        <!-- Interpolation: display component data -->
        <td>{{product.name}}</td>
        <td>{{product.price | currency}}</td>

        <!-- Property binding: set button disabled state from component -->
        <td><button [disabled]="product.stock === 0" (click)="addToCart(product)">
          {{product.stock === 0 ? 'Out of Stock' : 'Add to Cart'}}
        </button></td>
      </tr>
    </table>
  `
})
export class ProductListComponent {
  allProducts: Product[] = [];
  filteredProducts: Product[] = [];
  searchText: string = '';  // ← bound to [(ngModel)]

  constructor(private productService: ProductService) { }

  ngOnInit(): void {
    this.productService.getAll().subscribe(
      (data) => {
        this.allProducts = data;
        this.filteredProducts = data;
      }
    );
  }

  // Event handler — (click)="onSearch()" calls this
  onSearch(): void {
    this.filteredProducts = this.allProducts.filter(
      (p) => p.name.toLowerCase().includes(this.searchText.toLowerCase())
    );
  }

  // Event handler — (click)="addToCart(product)" calls this
  addToCart(product: Product): void {
    console.log('Added to cart:', product.name);
    // Update component state, call service, etc.
  }
}

    Data flow: Component → Template (property binding, interpolation); Template → Component (event binding, two-way binding). Angular watches for changes and updates the DOM automatically (change detection).

    Q5. What are Observables and RxJS? How do you handle async data? ▼

    Observable is a lazy stream of values over time. An HTTP request returns an Observable. An Observable does NOTHING until someone subscribes to it. When subscribed, it emits values (success, error, or completion).

    RxJS (Reactive Extensions for JavaScript) provides Observables and operators (map, filter, switchMap, etc.) to transform data streams in a declarative way.

    typescript — Observables and RxJS operators 
    // product.service.ts — returns Observables
import { Observable, of, throwError } from 'rxjs';
import { map, filter, switchMap, catchError, tap } from 'rxjs/operators';

@Injectable({ providedIn: 'root' })
export class ProductService {
  constructor(private http: HttpClient) { }

  // getAll returns an Observable — it does nothing until subscribed
  getAll(): Observable<Product[]> {
    return this.http.get<Product[]>('/api/products');
  }

  // Chaining operators: transform the stream before consumer receives it
  getExpensiveProducts(): Observable<Product[]> {
    return this.getAll().pipe(
      // tap: side effect (e.g., logging) without changing the data
      tap((products) => console.log(`Fetched ${products.length} products`)),

      // map: transform each item
      map((products) =>
        products.map((p) => ({
          ...p,
          discountedPrice: p.price * 0.9  // 10% off
        }))
      ),

      // filter: keep only items matching a condition
      filter((p) => p.price > 100),

      // catchError: handle errors gracefully
      catchError((error) => {
        console.error('Error fetching products:', error);
        return throwError(() => new Error('Failed to load products'));
      })
    );
  }

  getByCategory(category: string): Observable<Product[]> {
    return this.http.get<Product[]>(`/api/products/category/${category}`);
  }
}

// ─────────────────────────────────────────────────────────

// product-list.component.ts — subscribe to Observables
export class ProductListComponent implements OnInit {
  products: Product[] = [];
  loading: boolean = false;
  error: string | null = null;

  constructor(private productService: ProductService) { }

  ngOnInit(): void {
    this.loading = true;

    // SUBSCRIBE: Observable starts executing when subscribed
    this.productService.getExpensiveProducts().subscribe({
      next: (data) => {
        this.products = data;        // on success, update component
        this.loading = false;
      },
      error: (err) => {
        this.error = err.message;    // on error, set error state
        this.loading = false;
      },
      complete: () => {
        console.log('Data stream complete'); // optional
      }
    });
  }
}

// ─────────────────────────────────────────────────────────

// TEMPLATE — async pipe automatically subscribes/unsubscribes
@Component({
  template: `
    <div *ngIf="loading">Loading...</div>
    <div *ngIf="error">Error: {{error}}</div>
    <div *ngIf="!loading && !error">
      <tr *ngFor="let p of products">
        <td>{{p.name}}</td>
        <td>{{p.price}}</td>
      </tr>
    </div>
  `
})
export class ProductListComponent { ... }

// ─────────────────────────────────────────────────────────

// OR use async pipe in template — even cleaner
@Component({
  template: `
    <!-- async pipe automatically subscribes/unsubscribes -->
    <div *ngIf="(products$ | async) as products">
      <tr *ngFor="let p of products">
        <td>{{p.name}}</td>
      </tr>
    </div>
  `
})
export class ProductListComponent {
  products$: Observable<Product[]>;  // Observable, not array

  constructor(private productService: ProductService) {
    this.products$ = this.productService.getAll();
  }
}

    Key differences: Observables are lazy (do nothing until subscribed); Promises are eager (execute immediately). Observables can emit multiple values; Promises emit one. Observables are cancellable; Promises are not.

    Best practice: Use the async pipe in templates whenever possible — it automatically handles subscription/unsubscription and avoids memory leaks.

    Q6. What are Directives? ▼

    Directives are classes that add behavior to HTML elements or components. There are two types:

    Structural directives change the DOM structure — they add/remove elements. Syntax: *directive (asterisk prefix). Examples: *ngIf, *ngFor, *ngSwitch.

    Attribute directives change appearance or behavior of an element. Syntax: [directive] (bracket notation). Examples: [ngClass], [ngStyle], [routerLink].

    html — Structural and Attribute Directives 
    <!-- ──────────── STRUCTURAL DIRECTIVES ──────────── -->

<!-- *ngIf: conditionally render -->
<div *ngIf="products.length > 0">
  <p>{{products.length}} products available</p>
</div>
<div *ngIf="products.length === 0">
  <p>No products found</p>
</div>

<!-- *ngFor: loop over array -->
<table>
  <tr *ngFor="let product of products; let i = index; let last = last">
    <td>{{i + 1}}</td>
    <td>{{product.name}}</td>
    <td>{{product.price | currency}}</td>
    <!-- last is true for last item -->
    <td *ngIf="last">(End of list)</td>
  </tr>
</table>

<!-- *ngSwitch: multi-way conditional -->
<div [ngSwitch]="product.category">
  <div *ngSwitchCase="'Electronics'">
    <p>Electronics: warranty included</p>
  </div>
  <div *ngSwitchCase="'Clothing'">
    <p>Clothing: free returns</p>
  </div>
  <div *ngSwitchDefault>
    <p>Other category</p>
  </div>
</div>

<!-- ──────────── ATTRIBUTE DIRECTIVES ──────────── -->

<!-- [ngClass]: conditionally apply CSS classes -->
<div [ngClass]="{
  'in-stock': product.stock > 0,
  'out-of-stock': product.stock === 0,
  'expensive': product.price > 500
}">
  {{product.name}}
</div>

<!-- [ngStyle]: conditionally apply inline styles -->
<div [ngStyle]="{
  'color': product.stock > 0 ? 'green' : 'red',
  'font-weight': product.price > 1000 ? 'bold' : 'normal'
}">
  Price: {{product.price}}
</div>

<!-- [routerLink]: navigation without page reload -->
<a [routerLink]="['/product', product.id]">
  View Details
</a>

<!-- Custom directive example: [appHighlight] -->
<p [appHighlight]="'yellow'">This text has yellow background</p>
    typescript — Custom Attribute Directive 
    // highlight.directive.ts
import { Directive, ElementRef, Input, OnInit } from '@angular/core';

@Directive({
  selector: '[appHighlight]'  // usage: <p appHighlight="yellow">
})
export class HighlightDirective implements OnInit {
  @Input() appHighlight: string = 'yellow';  // color passed in

  constructor(private el: ElementRef) { }

  ngOnInit(): void {
    this.el.nativeElement.style.backgroundColor = this.appHighlight;
  }
}

    Common mistake: You cannot combine multiple structural directives on the same element. Use ng-container to avoid this:

    <ng-container *ngIf="condition"> <div *ngFor="let item of items">...</div> </ng-container>
    Q7. How do services handle data sharing between components? ▼

    Services are the primary way components communicate without tight coupling. A shared service holds state and logic; multiple components inject it and access/modify that shared state. The service can also provide Observables for reactive data sharing.

    typescript — Service as shared state holder 
    // cart.service.ts — shared service for cart state
import { Injectable } from '@angular/core';
import { BehaviorSubject, Observable } from 'rxjs';

export interface CartItem {
  product: Product;
  quantity: number;
}

@Injectable({ providedIn: 'root' })
export class CartService {
  // BehaviorSubject = Observable that also holds current value
  private cartItems$ = new BehaviorSubject<CartItem[]>([]);

  // Expose Observable to components
  cartItems: Observable<CartItem[]> = this.cartItems$.asObservable();

  // Expose count as Observable
  cartCount$: Observable<number> = this.cartItems$.pipe(
    map((items) => items.reduce((sum, item) => sum + item.quantity, 0))
  );

  constructor() { }

  addToCart(product: Product, quantity: number = 1): void {
    const current = this.cartItems$.value;  // get current value
    const existing = current.find((item) => item.product.id === product.id);

    if (existing) {
      existing.quantity += quantity;  // increase quantity
    } else {
      current.push({ product, quantity });  // add new item
    }

    this.cartItems$.next([...current]);  // emit new value to all subscribers
  }

  removeFromCart(productId: number): void {
    const current = this.cartItems$.value;
    const filtered = current.filter((item) => item.product.id !== productId);
    this.cartItems$.next(filtered);
  }

  clearCart(): void {
    this.cartItems$.next([]);
  }
}

// ─────────────────────────────────────────────────────────

// product-list.component.ts — displays products, adds to cart
@Component({
  selector: 'app-product-list',

    <div *ngFor="let product of products">

  ) { }

      (data) => this.products = data

// ─────────────────────────────────────────────────────────

// cart.component.ts — displays cart contents
@Component({
  selector: 'app-cart',
  template: `
    <div class="cart">
      <h2>Cart ({{(cartCount$ | async) || 0}} items)</h2>
      <div *ngFor="let item of (cartItems$ | async) || []">
        <span>{{item.product.name}} x {{item.quantity}}</span>
        <button (click)="remove(item.product.id)">Remove</button>
      </div>
      <button (click)="clear()">Clear Cart</button>
    </div>
  `
})
export class CartComponent {
  cartItems$: Observable<CartItem[]>;
  cartCount$: Observable<number>;

  constructor(private cartService: CartService) {
    this.cartItems$ = this.cartService.cartItems;
    this.cartCount$ = this.cartService.cartCount$;
  }

  remove(productId: number): void {
    this.cartService.removeFromCart(productId);
  }

  clear(): void {
    this.cartService.clearCart();
  }
}

    BehaviorSubject is a special Observable that emits its current value immediately to new subscribers and remembers the last emitted value. Perfect for shared state. Regular Observable emits only on new subscriptions.

    Q8. How does routing work in Angular? ▼

    Angular Router enables client-side navigation — changing the URL and displaying different components without a page reload (SPA behavior). Routes map URL paths to components. Navigation updates the URL and renders the matched component in <router-outlet>.

    typescript — Routing configuration for Product Catalog 
    // app-routing.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { ProductListComponent } from './products/product-list.component';
import { ProductDetailComponent } from './products/product-detail.component';
import { CartComponent } from './cart/cart.component';
import { CheckoutComponent } from './checkout/checkout.component';

// Define routes
const routes: Routes = [
  { path: '', redirectTo: '/products', pathMatch: 'full' },  // home redirects to /products
  { path: 'products', component: ProductListComponent },
  { path: 'product/:id', component: ProductDetailComponent },  // :id is a route parameter
  { path: 'cart', component: CartComponent },
  { path: 'checkout', component: CheckoutComponent },
  { path: '**', redirectTo: '/products' }  // catch-all for 404
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

// ─────────────────────────────────────────────────────────

// app.component.ts — root component
@Component({
  selector: 'app-root',
  template: `
    <nav>
      <a routerLink="/products" routerLinkActive="active">Products</a>
      <a routerLink="/cart" routerLinkActive="active">Cart</a>
    </nav>
    <!-- Router-outlet: displays matched component based on current URL -->
    <router-outlet></router-outlet>
  `
})
export class AppComponent { }

// ─────────────────────────────────────────────────────────

// product-detail.component.ts — access route parameters
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-product-detail',
  template: `
    <div *ngIf="product">
      <h1>{{product.name}}</h1>
      <p>{{product.description}}</p>
      <p>Price: {{product.price | currency}}</p>
      <button (click)="onAddToCart()">Add to Cart</button>
      <button (click)="goBack()">Back to Products</button>
    </div>
  `
})
export class ProductDetailComponent implements OnInit {
  product: Product | null = null;

  constructor(
    private route: ActivatedRoute,
    private productService: ProductService,
    private router: Router
  ) { }

  ngOnInit(): void {
    // Access route parameter :id
    this.route.paramMap.subscribe((params) => {
      const id = +params.get('id')!;  // + converts string to number
      this.productService.getById(id).subscribe(
        (data) => this.product = data,
        (error) => console.error('Product not found')
      );
    });
  }

  onAddToCart(): void {
    if (this.product) {
      // ... add to cart logic
      alert(`Added ${this.product.name} to cart`);
    }
  }

  goBack(): void {
    this.router.navigate(['/products']);  // programmatic navigation
  }
}

// Lazy loading — load feature module only when route accessed
const routes: Routes = [
  { path: 'products', component: ProductListComponent },
  {
    path: 'admin',
    loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule)
    // AdminModule is downloaded ONLY when user navigates to /admin
  }
];

    Key concepts: routerLink directive for navigation in templates; ActivatedRoute to access route params; Router.navigate() for programmatic navigation; lazy-loading for code splitting.

    Q9. What are Reactive Forms vs Template-driven Forms? ▼

    Template-driven forms keep form state in the template using [(ngModel)]. Simple, good for small forms. Validation is declared in the template.

    Reactive forms keep form state in the component class using FormBuilder and FormGroup. Powerful, testable, good for complex forms. Validation is logic-based.

    typescript — Template-driven form (simple) 
    // product-create.component.ts — TEMPLATE-DRIVEN
@Component({
  selector: 'app-product-create',
  template: `
    <form (ngSubmit)="onSubmit()">
      <label>
        Name:
        <input [(ngModel)]="product.name" name="name" required>
      </label>

      <label>
        Price:
        <input [(ngModel)]="product.price" name="price" type="number" min="0">
      </label>

      <label>
        Category:
        <select [(ngModel)]="product.category" name="category">
          <option value="">Select category</option>
          <option value="Electronics">Electronics</option>
          <option value="Clothing">Clothing</option>
        </select>
      </label>

      <button type="submit">Create Product</button>
    </form>
  `
})
export class ProductCreateComponent {
  product = { name: '', price: 0, category: '' };

  constructor(private productService: ProductService) { }

  onSubmit(): void {
    this.productService.create(this.product).subscribe(
      (result) => console.log('Created:', result)
    );
  }
}
    typescript — Reactive form (powerful) 
    // product-create.component.ts — REACTIVE
import { FormBuilder, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-product-create',
  template: `
    <form [formGroup]="form" (ngSubmit)="onSubmit()">
      <div>
        <label>Name:</label>
        <input formControlName="name">
        <span *ngIf="form.get('name')?.hasError('required')">
          Name is required
        </span>
      </div>

      <div>
        <label>Price:</label>
        <input type="number" formControlName="price">
        <span *ngIf="form.get('price')?.hasError('min')">
          Price must be >= 0
        </span>
      </div>

      <div>
        <label>Category:</label>
        <select formControlName="category">
          <option value="">Select category</option>
          <option value="Electronics">Electronics</option>
          <option value="Clothing">Clothing</option>
        </select>
      </div>

      <!-- Form is valid only if all controls pass validation -->
      <button [disabled]="form.invalid">Create Product</button>
    </form>
  `
})
export class ProductCreateComponent implements OnInit {
  form: FormGroup;

  constructor(
    private fb: FormBuilder,
    private productService: ProductService
  ) {
    // FormBuilder creates FormGroup with controls and validators
    this.form = this.fb.group({
      name: ['', [Validators.required, Validators.minLength(3)]],
      price: [0, [Validators.required, Validators.min(0)]],
      category: ['', Validators.required]
    });
  }

  ngOnInit(): void {
    // Watch for form changes
    this.form.valueChanges.subscribe((value) => {
      console.log('Form changed:', value);
    });
  }

  onSubmit(): void {
    if (this.form.invalid) return;

    this.productService.create(this.form.value).subscribe(
      (result) => {
        console.log('Created:', result);
        this.form.reset();  // clear form after submit
      }
    );
  }
}
    AspectTemplate-drivenReactive
    State locationTemplate (HTML)Component (TypeScript)
    ComplexitySimple, few controlsComplex, many controls
    TestabilityHarder (tied to template)Easy (logic in component)
    ValidationDeclarative attributesProgrammable Validators
    Dynamic controlsDifficultFormArray makes it easy

    Recommendation: Use Reactive Forms for production apps — they are testable, powerful, and scale with complexity.

    Angular Lifecycle Hooks

    Angular calls lifecycle hook methods at specific moments in a component's lifecycle — from creation, through updates, to destruction. Each hook is an opportunity to run code at the right time.

    Hook When Called Use Case Example
    ngOnInit After component class created, after first @Input bindings initialized Load initial data, subscribe to services, set up component state this.productService.getAll().subscribe(...)
    ngOnChanges Before ngOnInit, and when @Input property value changes React to input property changes (parent passes new data) When @Input() productId changes, reload product details
    ngDoCheck During every change detection cycle (frequent!) Custom change detection logic (rarely needed) Detect changes Angular's default detection misses
    ngAfterViewInit After component view and child views initialized (@ViewChild available) Access DOM elements, initialize third-party libraries (jQuery plugins, charts) this.chart.initialize(this.chartElement.nativeElement)
    ngOnDestroy Just before component is destroyed and removed from DOM Cleanup: unsubscribe from Observables, clear timers, release resources this.subscription.unsubscribe()
    typescript — Lifecycle hooks in ProductDetailComponent 
    import {
  Component,
  OnInit,
  OnChanges,
  OnDestroy,
  AfterViewInit,
  Input,
  ViewChild,
  SimpleChanges,
  ElementRef
} from '@angular/core';
import { Subscription } from 'rxjs';

@Component({
  selector: 'app-product-detail',
  template: `
    <div>
      <h1>{{product?.name}}</h1>
      <p>Price: {{product?.price}}</p>
      <div #chartContainer></div>  <!-- @ViewChild access this -->
    </div>
  `
})
export class ProductDetailComponent implements OnInit, OnChanges, AfterViewInit, OnDestroy {
  @Input() productId: number = 0;
  product: Product | null = null;
  @ViewChild('chartContainer') chartContainer!: ElementRef;
  private subscription: Subscription | null = null;

  constructor(private productService: ProductService) {
    console.log('1. CONSTRUCTOR — component instance created (NO ACCESS to @Input yet)');
  }

  // ──────────────────────────────────────────────────────

  ngOnChanges(changes: SimpleChanges): void {
    console.log('2. ngOnChanges — @Input properties changed');
    console.log('   Changes:', changes);

    if (changes['productId']) {
      console.log(`   productId changed from ${changes['productId'].previousValue} to ${changes['productId'].currentValue}`);
      // Reload product when productId @Input changes
      if (!changes['productId'].firstChange) {
        this.loadProduct();
      }
    }
  }

  // ──────────────────────────────────────────────────────

  ngOnInit(): void {
    console.log('3. ngOnInit — @Input bindings ready, load initial data');
    console.log(`   productId = ${this.productId}`);

    // Load product on component init
    this.loadProduct();

    // Subscribe to route changes (if using router)
    // this.route.paramMap.subscribe(...);
  }

  // ──────────────────────────────────────────────────────

  ngAfterViewInit(): void {
    console.log('4. ngAfterViewInit — view rendered, @ViewChild available');

    // @ViewChild is NOW available
    if (this.chartContainer) {
      console.log('   Chart container element:', this.chartContainer.nativeElement);
      // Initialize third-party chart library
      // const chart = new ChartLibrary(this.chartContainer.nativeElement);
    }
  }

  // ──────────────────────────────────────────────────────

  ngOnDestroy(): void {
    console.log('5. ngOnDestroy — component about to be destroyed, cleanup');

    // Unsubscribe from Observables to prevent memory leaks
    if (this.subscription) {
      this.subscription.unsubscribe();
      console.log('   Unsubscribed from product service');
    }

    // Clear timers, event listeners, etc.
  }

  // ──────────────────────────────────────────────────────

  private loadProduct(): void {
    console.log(`   Loading product ${this.productId}...`);

    // Unsubscribe previous subscription if exists
    if (this.subscription) {
      this.subscription.unsubscribe();
    }

    // Subscribe to product service
    this.subscription = this.productService.getById(this.productId).subscribe(
      (data) => {
        this.product = data;
        console.log('   Product loaded:', data);
      },
      (error) => {
        console.error('   Error loading product:', error);
        this.product = null;
      }
    );
  }
}

// 1. CONSTRUCTOR — component instance created (NO ACCESS to @Input yet)
// 2. ngOnChanges — @Input properties changed
//    productId changed from 0 to 5
// 3. ngOnInit — @Input bindings ready, load initial data
//    productId = 5
//    Loading product 5...
//    Product loaded: { id: 5, name: 'Laptop', price: 999 }
// 4. ngAfterViewInit — view rendered, @ViewChild available
//    Chart container element: <div>...</div>
//
// (user navigates away or component removed)
//
// 5. ngOnDestroy — component about to be destroyed, cleanup
//    Unsubscribed from product service
    ⚠️

    Memory leak trap: forgetting to unsubscribe

    In ngOnDestroy, always unsubscribe from long-lived Observables. If you don't, the subscription stays active even after the component is destroyed, leaking memory. Use the async pipe in templates (it auto-unsubscribes) or implement OnDestroy and call .unsubscribe().
