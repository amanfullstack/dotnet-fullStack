🔴 Redux - Predictable State Management

Master Redux for managing complex application state across multiple components without prop drilling.

What is Redux?

Redux is a JavaScript library for managing application state in a predictable way. Instead of each component managing its own state, Redux creates a single source of truth (the store) where all state lives. Components dispatch actions to update state, triggering automatic UI updates.

💡

Key Idea: Redux replaces "prop drilling" with a centralized state store. No more passing data through 5 levels of components!

Redux vs Context API vs Local State

Solution Best For Complexity Performance
Local useState Simple component state (forms, toggles) Low Fast (minimal re-renders)
Context API Shared state (theme, auth, small datasets) Medium Good (but whole tree re-renders)
Redux Complex state, many components, time-travel debugging High Excellent (selective re-renders)

Redux Architecture: The Three Pillars

1. Actions - "What happened?"

Actions are plain objects that describe what happened. They MUST have a type property.

// Action (simple object)
{
  type: 'ADD_PRODUCT',
  payload: { id: 1, name: 'Widget', price: 9.99 }
}

// Action Creator (function that returns action)
const addProduct = (product) => ({
  type: 'ADD_PRODUCT',
  payload: product
});

2. Reducers - "How does state change?"

Pure functions that take current state and an action, return new state. Never mutate state directly!

// Reducer function
const productReducer = (state = [], action) => {
  switch(action.type) {
    case 'ADD_PRODUCT':
      // ❌ Wrong: mutating state
      // state.push(action.payload);

      // ✅ Correct: return new array
      return [...state, action.payload];

    case 'REMOVE_PRODUCT':
      return state.filter(p => p.id !== action.payload);

    default:
      return state;
  }
}

3. Store - "Where does state live?"

Single centralized store holding the entire application state. Created from reducers.

import { createStore } from 'redux';

// Combine reducers if multiple
import { combineReducers } from 'redux';

const rootReducer = combineReducers({
  products: productReducer,
  filter: filterReducer,
  user: userReducer
});

// Create store from reducer
const store = createStore(rootReducer);

// Store has methods:
store.getState();           // Get current state
store.dispatch(action);     // Send action
store.subscribe(listener);  // Listen for changes

Redux Flow Diagram

1. USER INTERACTION
   └─> User clicks "Add to Cart" button

2. ACTION DISPATCHED
   └─> dispatch({ type: 'ADD_TO_CART', payload: product })

3. REDUCER PROCESSES
   └─> cartReducer receives action
   └─> Returns new state with product added

4. STORE UPDATED
   └─> state.cart = [...state.cart, product]

5. SUBSCRIBERS NOTIFIED
   └─> All components using cart state are notified

6. COMPONENTS RE-RENDER
   └─> Only components using changed state re-render
   └─> UI shows updated cart count

Real-World Example: ProductCatalog with Redux

Step 1: Define Actions

// actions/productActions.js
export const FETCH_PRODUCTS = 'FETCH_PRODUCTS';
export const FETCH_PRODUCTS_SUCCESS = 'FETCH_PRODUCTS_SUCCESS';
export const FETCH_PRODUCTS_ERROR = 'FETCH_PRODUCTS_ERROR';
export const ADD_TO_CART = 'ADD_TO_CART';
export const REMOVE_FROM_CART = 'REMOVE_FROM_CART';

// Action creators
export const fetchProductsSuccess = (products) => ({
  type: FETCH_PRODUCTS_SUCCESS,
  payload: products
});

export const addToCart = (product) => ({
  type: ADD_TO_CART,
  payload: product
});

export const removeFromCart = (productId) => ({
  type: REMOVE_FROM_CART,
  payload: productId
});

Step 2: Create Reducers

// reducers/productReducer.js
import {
  FETCH_PRODUCTS_SUCCESS,
  FETCH_PRODUCTS_ERROR
} from '../actions/productActions';

const initialState = {
  items: [],
  loading: false,
  error: null
};

export const productReducer = (state = initialState, action) => {
  switch(action.type) {
    case FETCH_PRODUCTS_SUCCESS:
      return {
        ...state,
        items: action.payload,
        loading: false,
        error: null
      };

    case FETCH_PRODUCTS_ERROR:
      return {
        ...state,
        error: action.payload,
        loading: false
      };

    default:
      return state;
  }
};

// reducers/cartReducer.js
import { ADD_TO_CART, REMOVE_FROM_CART } from '../actions/productActions';

const initialState = {
  items: [],
  total: 0
};

export const cartReducer = (state = initialState, action) => {
  switch(action.type) {
    case ADD_TO_CART:
      // Check if product already in cart
      const existing = state.items.find(p => p.id === action.payload.id);

      if (existing) {
        // Increase quantity
        return {
          ...state,
          items: state.items.map(p =>
            p.id === action.payload.id
              ? { ...p, quantity: p.quantity + 1 }
              : p
          ),
          total: state.total + action.payload.price
        };
      } else {
        // Add new product
        return {
          ...state,
          items: [...state.items, { ...action.payload, quantity: 1 }],
          total: state.total + action.payload.price
        };
      }

    case REMOVE_FROM_CART:
      const removed = state.items.find(p => p.id === action.payload);
      return {
        ...state,
        items: state.items.filter(p => p.id !== action.payload),
        total: state.total - (removed?.price || 0)
      };

    default:
      return state;
  }
};

Step 3: Create Store

// store.js
import { createStore, combineReducers } from 'redux';
import { productReducer } from './reducers/productReducer';
import { cartReducer } from './reducers/cartReducer';

const rootReducer = combineReducers({
  products: productReducer,
  cart: cartReducer
});

const store = createStore(rootReducer);

export default store;

Step 4: Connect React with react-redux

// App.js
import React from 'react';
import { Provider } from 'react-redux';
import store from './store';
import ProductList from './components/ProductList';
import Cart from './components/Cart';

function App() {
  return (
    <Provider store={store}>
      <div className="app">
        <h1>ProductCatalog</h1>
        <ProductList />
        <Cart />
      </div>
    </Provider>
  );
}

export default App;

Step 5: Use Redux in Components

// components/ProductList.js
import { useSelector, useDispatch } from 'react-redux';
import { addToCart } from '../actions/productActions';

function ProductList() {
  // Read from Redux store
  const { items, loading, error } = useSelector(state => state.products);
  const dispatch = useDispatch();

  const handleAddToCart = (product) => {
    // Dispatch action to Redux
    dispatch(addToCart(product));
  };

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div className="product-list">
      {items.map(product => (
        <div key={product.id} className="product-card">
          <h3>{product.name}</h3>
          <p>${product.price}</p>
          <button onClick={() => handleAddToCart(product)}>
            Add to Cart
          </button>
        </div>
      ))}
    </div>
  );
}

// components/Cart.js
import { useSelector, useDispatch } from 'react-redux';
import { removeFromCart } from '../actions/productActions';

function Cart() {
  const { items, total } = useSelector(state => state.cart);
  const dispatch = useDispatch();

  return (
    <div className="cart">
      <h2>Shopping Cart ({items.length} items)</h2>

      {items.map(item => (
        <div key={item.id} className="cart-item">
          <span>{item.name} x{item.quantity}</span>
          <span>${(item.price * item.quantity).toFixed(2)}</span>
          <button onClick={() => dispatch(removeFromCart(item.id))}>
            Remove
          </button>
        </div>
      ))}

      <h3>Total: ${total.toFixed(2)}</h3>
    </div>
  );
}

Handling Async Operations with Middleware

Middleware lets you handle async operations (API calls) in Redux. Popular choices: redux-thunk or redux-saga.

Using Redux Thunk

// Setup with thunk middleware
import { createStore, combineReducers, applyMiddleware } from 'redux';
import thunk from 'redux-thunk';

const store = createStore(rootReducer, applyMiddleware(thunk));

// Action that calls API
export const fetchProducts = () => {
  return async (dispatch) => {
    try {
      // Call API
      const response = await fetch('http://localhost:5000/api/products');
      const data = await response.json();

      // Dispatch success action
      dispatch({
        type: FETCH_PRODUCTS_SUCCESS,
        payload: data
      });
    } catch (error) {
      // Dispatch error action
      dispatch({
        type: FETCH_PRODUCTS_ERROR,
        payload: error.message
      });
    }
  };
};

// Use in component
function ProductList() {
  const dispatch = useDispatch();

  useEffect(() => {
    // Dispatch thunk action
    dispatch(fetchProducts());
  }, [dispatch]);
}

Best Practices

  1. Keep Reducers Pure: No side effects, same input = same output
  2. Normalize State Shape: Store data by ID for easy updates 
    // ❌ Wrong - nested data
{ products: [{ id: 1, reviews: [...] }] }

// ✅ Correct - normalized
{
  products: { 1: { id: 1, name: '...' } },
  reviews: { 1: { id: 1, productId: 1, text: '...' } }
}
  3. Use Selectors: Avoid accessing state directly 
    // ❌ Wrong
const cart = useSelector(state => state.cart.items);

// ✅ Better - create selector
export const selectCartItems = (state) => state.cart.items;
const cart = useSelector(selectCartItems);
  4. Split by Feature: Organize reducers by domain (products, cart, auth)
  5. Use Redux DevTools: Debug state changes with time-travel

Interview Q&A

Q1: What are the benefits of Redux over local state?

Answer:

  • Single Source of Truth: All state in one store, easier to debug
  • No Prop Drilling: Access state from any component directly, no need to pass through 5 parent components
  • Time-Travel Debugging: Redux DevTools let you step backward/forward through state changes
  • Predictable: Pure reducers make behavior predictable and testable
  • Performance: Only components using changed state re-render

Q2: When should you use Redux vs Context API?

Answer:

  • Use Context API: Global theme, auth status, or small datasets that rarely change
  • Use Redux: Complex state (shopping cart, filters, user preferences), frequent updates, many nested components
  • Rule of Thumb: Start with Context API. Switch to Redux if prop drilling becomes painful

Q3: What's the difference between actions and reducers?

Answer:

  • Actions: Objects describing "what happened" (ADD_PRODUCT, REMOVE_PRODUCT). Must have type.
  • Reducers: Pure functions that calculate "how state changes" given action. Input: (state, action) => newState
  • Analogy: Action = event description, Reducer = business logic that responds to event
