🔄 Custom Hooks

Create reusable logic with custom hooks - extract common patterns into shareable functions.

What Are Custom Hooks?

Custom hooks are JavaScript functions that use React hooks (like useState, useEffect) to extract component logic into reusable units. They follow the naming convention use* and let you share stateful logic between components without render props or higher-order components.

Think of custom hooks as utilities that encapsulate complex logic - similar to how services work in Angular!

Why Custom Hooks Matter

  • Code Reuse: Share logic between components without duplicating code
  • Separation of Concerns: Keep component logic and UI separate
  • Testability: Test hook logic independently from components
  • Cleaner Components: Move complex logic out of component JSX
  • Team Standards: Build shared utilities that entire team uses

Key Concepts

📋 Hook Naming Convention

  • Custom hooks MUST start with use (e.g., useFetch, useForm)
  • React's linter enforces this rule
  • Tells React and developers this function uses hooks

🏗️ Hook Structure

Custom hooks are just functions that call React hooks:

function useMyHook(params) {
  // Can use any React hooks here
  const [state, setState] = useState(...);
  useEffect(() => { ... }, []);

  // Return whatever you want
  return { state, setState, someFunction };
}

📦 What to Return

Return an object or array of values:

// Return object (named access)
return { data, loading, error };

// Return array (positional access)
return [data, loading];

// Return function
return updateData;

🔄 Hook Composition

Custom hooks can use other hooks, including other custom hooks:

// Hook using useState and useEffect
function useFetch(url) {
  const [data, setData] = useState(null);
  useEffect(() => { ... }, [url]);
  return data;
}

// Hook using another hook
function useUser(userId) {
  const data = useFetch(`/api/users/${userId}`);
  return data;
}

Code Examples

Example 1: useFetch Hook (API Calls)

import { useState, useEffect } from 'react';

// Custom hook for fetching data
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        setLoading(true);
        const response = await fetch(url);
        if (!response.ok) throw new Error('Network error');
        const json = await response.json();
        setData(json);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, [url]); // Re-fetch if URL changes

  return { data, loading, error };
}

// Use in component
function ProductList() {
  const { data: products, loading, error } = useFetch('http://localhost:5000/api/products');

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {products?.map(p => <li key={p.id}>{p.name}</li>)}
    </ul>
  );
}

Example 2: useLocalStorage Hook

import { useState, useEffect } from 'react';

// Hook for managing localStorage
function useLocalStorage(key, initialValue) {
  // Get from localStorage or use initial value
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  // Update localStorage when state changes
  const setValue = (value) => {
    try {
      const valueToStore = value instanceof Function ? value(storedValue) : value;
      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  };

  return [storedValue, setValue];
}

// Use in component
function UserPreferences() {
  const [theme, setTheme] = useLocalStorage('theme', 'light');
  const [fontSize, setFontSize] = useLocalStorage('fontSize', 16);

  return (
    <div>
      <label>
        Theme:
        <select value={theme} onChange={e => setTheme(e.target.value)}>
          <option>light</option>
          <option>dark</option>
        </select>
      </label>

      <label>
        Font Size:
        <input
          type="number"
          value={fontSize}
          onChange={e => setFontSize(parseInt(e.target.value))}
        />
      </label>

      <!-- Settings persist across page reloads! -->
    </div>
  );
}

Example 3: useForm Hook

import { useState, useCallback } from 'react';

// Hook for form handling
function useForm(initialValues, onSubmit) {
  const [values, setValues] = useState(initialValues);
  const [errors, setErrors] = useState({});

  const handleChange = useCallback((e) => {
    const { name, value } = e.target;
    setValues(prev => ({ ...prev, [name]: value }));
    // Clear error for this field
    setErrors(prev => ({ ...prev, [name]: '' }));
  }, []);

  const handleSubmit = useCallback((e) => {
    e.preventDefault();
    onSubmit(values);
  }, [values, onSubmit]);

  const reset = useCallback(() => {
    setValues(initialValues);
    setErrors({});
  }, [initialValues]);

  return {
    values,
    errors,
    setErrors,
    handleChange,
    handleSubmit,
    reset
  };
}

// Use in component
function LoginForm() {
  const { values, errors, handleChange, handleSubmit } = useForm(
    { email: '', password: '' },
    async (values) => {
      try {
        // API call
        const response = await fetch('/api/login', {
          method: 'POST',
          body: JSON.stringify(values)
        });
        if (!response.ok) throw new Error('Login failed');
        // Success!
      } catch (err) {
        // Handle error
      }
    }
  );

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="email"
        value={values.email}
        onChange={handleChange}
        placeholder="Email"
      />
      {errors.email && <span>{errors.email}</span>}

      <input
        name="password"
        type="password"
        value={values.password}
        onChange={handleChange}
        placeholder="Password"
      />
      {errors.password && <span>{errors.password}</span>}

      <button type="submit">Login</button>
    </form>
  );
}

Example 4: useWindowSize Hook

import { useState, useEffect } from 'react';

// Hook to track window dimensions
function useWindowSize() {
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight
  });

  useEffect(() => {
    const handleResize = () => {
      setSize({
        width: window.innerWidth,
        height: window.innerHeight
      });
    };

    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);

  return size;
}

// Use in component
function ResponsiveLayout() {
  const { width, height } = useWindowSize();

  return (
    <div>
      <p>Window: {width}x{height}</p>
      <p>{width < 768 ? 'Mobile' : 'Desktop'} view</p>
    </div>
  );
}

✅ Best Practices

1. Always Name Hooks with "use" Prefix

Why: React's linter and developers will understand these are hooks

// ✅ Good
function useFetch() { }
function useForm() { }
function useLocalStorage() { }

// ❌ Wrong - not recognizable as hook
function fetchData() { }
function getFormState() { }

2. Keep Hooks Simple and Focused

Why: Each hook should do one thing well

❌ Bad: One hook that handles form, validation, API, and localization

✅ Good: useForm for form logic, useValidation for validation, useFetch for API

3. Return Consistent Shapes

Why: Predictable API makes hooks easier to use

// ✅ Consistent: always return object with meaningful keys
return {
  data: products,
  loading: isLoading,
  error: errorMessage,
  refetch: () => { }
};

4. Include Proper Dependencies in useEffect

Why: Avoids stale data and infinite loops

// Any variables from hook params should be in dependencies
useEffect(() => {
  fetch(url);
}, [url]); // Include url if it's from params

5. Document Your Hooks

Why: Other developers need to understand how to use them

/**
 * Fetches data from API
 * @param {string} url - API endpoint
 * @returns {{ data, loading, error }}
 */
function useFetch(url) {
  // ...
}

❌ Common Mistakes

1. Forgetting to Handle Loading State

// ❌ Wrong - components don't know when data is loading
return { data };

// ✅ Correct
return { data, loading, error };

2. Using Hooks Inside Conditions

// ❌ Wrong - hook order becomes inconsistent
function useConditionalHook(condition) {
  if (condition) {
    useState(...); // Hook inside condition!
  }
}

// ✅ Correct - always call hooks at top level
function useConditionalLogic(condition) {
  const [value, setValue] = useState(null);
  // Conditional logic inside hook
  if (condition) {
    // Do something
  }
}

3. Not Handling Cleanup

Especially important for event listeners, timers, subscriptions

// ❌ Wrong - listener never removed
useEffect(() => {
  window.addEventListener('resize', handleResize);
}, []);

// ✅ Correct - cleanup removes listener
useEffect(() => {
  window.addEventListener('resize', handleResize);
  return () => window.removeEventListener('resize', handleResize);
}, []);

🎓 Interview Q&A

Q1: What's the difference between a custom hook and a utility function?

Answer:

  • Custom Hook: Calls React hooks (useState, useEffect), must start with "use"
  • Utility Function: Regular JavaScript function, no hook calls
  • Use hooks for logic with state/side effects
  • Use utils for pure logic (formatting, calculations)

Q2: Why do custom hooks need to start with "use"?

Answer: The "use" prefix signals two important things:

  • React's linter can enforce hook rules
  • Other developers know this function uses hooks
  • Breaks the rules of hooks if used in unexpected ways

Q3: How do you share state between components using custom hooks?

Answer: Each component gets its own state instance when using custom hooks. To share state between components, use Context API:

// Each component gets own state
const CompA = () => {
  const [count, setCount] = useCounter(); // Count = 0
};

const CompB = () => {
  const [count, setCount] = useCounter(); // Count = 0 (separate)
};

// To share state, use Context + custom hook
const SharedCounterContext = createContext();

function useSharedCounter() {
  return useContext(SharedCounterContext);
}

// Now both components share the same state
