📋 Reactive Forms

Build powerful, testable forms with FormBuilder, validation, and reactive patterns.

What Are Reactive Forms?

Reactive Forms give you full programmatic control over form structure and validation. Instead of putting logic in templates, you write it in your TypeScript component using FormControl, FormGroup, and FormBuilder.

This approach is perfect for complex validation, dynamic fields, real-time form state management, and unit testing.

Why Reactive Forms Matter

  • Testability: Validation logic is in code, not templates - easy to test
  • Flexibility: Build forms dynamically with FormArray and nested FormGroups
  • Reactive: Subscribe to form changes with observables
  • Powerful Validation: Mix built-in and custom validators, sync and async
  • Type Safety: Work with strongly-typed form data

Key Concepts

🎮 FormControl

Represents a single form field. Tracks value, validation state, and touched/dirty status.

// Create a single control
const nameControl = new FormControl('', Validators.required);

// Check state
console.log(nameControl.value);        // Current value
console.log(nameControl.valid);        // Is it valid?
console.log(nameControl.touched);      // Has user interacted?
console.log(nameControl.dirty);        // Has value changed?

📦 FormGroup

Groups multiple FormControls together. Useful for complex forms with related fields.

// Group multiple controls
const form = new FormGroup({
  firstName: new FormControl(''),
  lastName: new FormControl(''),
  email: new FormControl('')
});

// Access individual controls
form.get('firstName').value;
form.get('email').hasError('email');

🔄 FormArray

Dynamic array of FormControls or FormGroups. Perfect for "Add More" features.

// Create array of addresses
const addressArray = new FormArray([
  new FormGroup({
    street: new FormControl(''),
    city: new FormControl('')
  })
]);

// Add more addresses dynamically
addressArray.push(new FormGroup({
  street: new FormControl(''),
  city: new FormControl('')
}));

🛠️ FormBuilder

Convenient shorthand for creating forms. Much less verbose than manual FormGroup/FormControl.

// Using FormBuilder (recommended)
this.form = this.fb.group({
  email: ['', [Validators.required, Validators.email]],
  password: ['', [Validators.required, Validators.minLength(8)]]
});

// Instead of manual creation:
// this.form = new FormGroup({
//   email: new FormControl('', [Validators.required, Validators.email]),
//   password: new FormControl('', [Validators.required, Validators.minLength(8)])
// });

✅ Validators

Built-in and custom validation functions:

  • Synchronous: Validators.required, email, minLength, pattern, etc.
  • Asynchronous: Check email availability, username uniqueness via API
  • Custom: Write your own ValidatorFn for specific business logic

Code Examples

Example 1: Basic Login Form

import { Component, OnInit } from '@angular/core';
import { FormBuilder, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-login',
  templateUrl: './login.component.html',
  styleUrls: ['./login.component.css']
})
export class LoginComponent implements OnInit {
  loginForm: FormGroup;

  constructor(private fb: FormBuilder) {}

  ngOnInit() {
    // Create form with validation
    this.loginForm = this.fb.group({
      email: ['', [Validators.required, Validators.email]],
      password: ['', [
        Validators.required,
        Validators.minLength(8)
      ]],
      rememberMe: [false] // Optional field
    });
  }

  // Helper getters for template
  get email() { return this.loginForm.get('email'); }
  get password() { return this.loginForm.get('password'); }

  onSubmit() {
    if (this.loginForm.valid) {
      console.log('Form Value:', this.loginForm.value);
      // {
      //   email: 'user@example.com',
      //   password: 'securePassword123',
      //   rememberMe: false
      // }
      // Call authentication service here
    }
  }

  reset() {
    this.loginForm.reset(); // Clear all values
  }
}

<!-- login.component.html -->
<form [formGroup]="loginForm" (ngSubmit)="onSubmit()">
  <!-- Email field -->
  <div class="form-group">
    <label for="email">Email</label>
    <input
      id="email"
      type="email"
      formControlName="email"
      placeholder="Enter your email"
      class="form-control">
    <!-- Show error only after user touches field -->
    <small class="error" *ngIf="email.invalid && email.touched">
      <span *ngIf="email.hasError('required')">Email is required</span>
      <span *ngIf="email.hasError('email')">Invalid email format</span>
    </small>
  </div>

  <!-- Password field -->
  <div class="form-group">
    <label for="password">Password</label>
    <input
      id="password"
      type="password"
      formControlName="password"
      placeholder="Minimum 8 characters"
      class="form-control">
    <small class="error" *ngIf="password.invalid && password.touched">
      <span *ngIf="password.hasError('required')">Password is required</span>
      <span *ngIf="password.hasError('minlength')">
        Password must be at least {{password.getError('minlength').requiredLength}} characters
      </span>
    </small>
  </div>

  <!-- Checkbox -->
  <div class="form-group">
    <input
      id="rememberMe"
      type="checkbox"
      formControlName="rememberMe">
    <label for="rememberMe">Remember me</label>
  </div>

  <!-- Submit button disabled until form is valid -->
  <button type="submit" [disabled]="loginForm.invalid">
    Login
  </button>
  <button type="button" (click)="reset()">Clear</button>
</form>

Example 2: Product Form with Dynamic Addresses (FormArray)

import { Component, OnInit } from '@angular/core';
import { FormBuilder, FormGroup, FormArray, Validators } from '@angular/forms';

@Component({
  selector: 'app-product-form',
  templateUrl: './product-form.component.html'
})
export class ProductFormComponent implements OnInit {
  productForm: FormGroup;

  constructor(private fb: FormBuilder) {}

  ngOnInit() {
    this.productForm = this.fb.group({
      productName: ['', [Validators.required, Validators.minLength(3)]],
      description: ['', Validators.required],
      price: ['', [Validators.required, Validators.min(0)]],
      // FormArray for multiple warehouse locations
      warehouses: this.fb.array([
        this.createWarehouseControl()
      ])
    });
  }

  // Helper to create a warehouse control
  private createWarehouseControl(): FormGroup {
    return this.fb.group({
      location: ['', Validators.required],
      quantity: ['', [Validators.required, Validators.min(1)]]
    });
  }

  // Getter for warehouses array
  get warehouses(): FormArray {
    return this.productForm.get('warehouses') as FormArray;
  }

  // Add new warehouse
  addWarehouse() {
    this.warehouses.push(this.createWarehouseControl());
  }

  // Remove warehouse at index
  removeWarehouse(index: number) {
    this.warehouses.removeAt(index);
  }

  onSubmit() {
    if (this.productForm.valid) {
      console.log('Product:', this.productForm.value);
      // {
      //   productName: 'Laptop',
      //   description: 'High-performance laptop',
      //   price: 1299,
      //   warehouses: [
      //     { location: 'NY', quantity: 50 },
      //     { location: 'CA', quantity: 30 }
      //   ]
      // }
    }
  }
}

<!-- product-form.component.html -->
<form [formGroup]="productForm" (ngSubmit)="onSubmit()">
  <!-- Product Name -->
  <div>
    <label>Product Name</label>
    <input formControlName="productName" type="text">
  </div>

  <!-- Description -->
  <div>
    <label>Description</label>
    <textarea formControlName="description"></textarea>
  </div>

  <!-- Price -->
  <div>
    <label>Price</label>
    <input formControlName="price" type="number">
  </div>

  <!-- Warehouses (FormArray) -->
  <div formArrayName="warehouses">
    <h3>Warehouse Locations</h3>
    <div *ngFor="let warehouse of warehouses.controls; let i = index" [formGroupName]="i">
      <div>
        <label>Location {{i + 1}}</label>
        <input formControlName="location" type="text" placeholder="e.g., New York">
      </div>
      <div>
        <label>Quantity</label>
        <input formControlName="quantity" type="number">
      </div>
      <button type="button" (click)="removeWarehouse(i)" *ngIf="warehouses.length > 1">
        Remove Location
      </button>
    </div>
  </div>

  <!-- Add warehouse button -->
  <button type="button" (click)="addWarehouse()">
    + Add Warehouse Location
  </button>

  <!-- Submit -->
  <button type="submit" [disabled]="productForm.invalid">Save Product</button>
</form>

Example 3: Custom Validator for Password Matching

import { Component, OnInit } from '@angular/core';
import { FormBuilder, FormGroup, Validators, AbstractControl, ValidationErrors, ValidatorFn } from '@angular/forms';

// Custom validator function
function passwordMatchValidator(): ValidatorFn {
  return (formGroup: AbstractControl): ValidationErrors | null => {
    const password = formGroup.get('password');
    const confirmPassword = formGroup.get('confirmPassword');

    if (!password || !confirmPassword) {
      return null;
    }

    // Compare passwords
    if (password.value === confirmPassword.value) {
      return null; // Valid
    } else {
      confirmPassword.setErrors({ passwordMismatch: true });
      return { passwordMismatch: true }; // Invalid
    }
  };
}

@Component({
  selector: 'app-signup',
  templateUrl: './signup.component.html'
})
export class SignupComponent implements OnInit {
  signupForm: FormGroup;

  constructor(private fb: FormBuilder) {}

  ngOnInit() {
    this.signupForm = this.fb.group({
      username: ['', [
        Validators.required,
        Validators.minLength(3),
        Validators.maxLength(20)
      ]],
      email: ['', [Validators.required, Validators.email]],
      password: ['', [
        Validators.required,
        Validators.minLength(8),
        // Custom: must have uppercase, lowercase, number
        this.passwordStrengthValidator()
      ]],
      confirmPassword: ['', Validators.required]
    }, {
      // Apply group-level validator
      validators: passwordMatchValidator()
    });
  }

  // Custom validator for password strength
  private passwordStrengthValidator(): ValidatorFn {
    return (control: AbstractControl): ValidationErrors | null => {
      const value = control.value;

      if (!value) {
        return null;
      }

      const hasUpperCase = /[A-Z]/.test(value);
      const hasLowerCase = /[a-z]/.test(value);
      const hasNumeric = /[0-9]/.test(value);

      const valid = hasUpperCase && hasLowerCase && hasNumeric;

      if (!valid) {
        return { weakPassword: true };
      }

      return null;
    };
  }

  onSubmit() {
    if (this.signupForm.valid) {
      console.log('New User:', this.signupForm.value);
    }
  }

  get password() { return this.signupForm.get('password'); }
  get confirmPassword() { return this.signupForm.get('confirmPassword'); }
}

✅ Best Practices

1. Use FormBuilder for Cleaner Code

Why: FormBuilder shorthand is much more readable than manual FormGroup/FormControl

❌ Bad: Creating controls manually with new FormControl everywhere

✅ Good: Use this.fb.group() to create entire form structure in one place

2. Show Errors Only After User Interaction

Why: Users get frustrated seeing errors before they finish typing

<!-- Only show error if field is invalid AND touched -->
<span *ngIf="email.invalid && email.touched" class="error">
  Invalid email
</span>

3. Use Helper Getters in Template

Why: Makes templates cleaner and easier to read

// In component
get email() { return this.form.get('email'); }
get password() { return this.form.get('password'); }

// In template - much cleaner
<span *ngIf="email.hasError('required')">Email required</span>

4. Disable Submit Button Until Form Valid

Why: Prevents invalid submissions and guides user

<button type="submit" [disabled]="form.invalid">
  Submit
</button>

5. Import ReactiveFormsModule in Your Module

Why: Reactive forms require this module to work

import { ReactiveFormsModule } from '@angular/forms';

@NgModule({
  imports: [ReactiveFormsModule],
  // ...
})
export class YourModule { }

6. Use Async Validators for API Calls

Why: Check email availability, username uniqueness without blocking form

email: ['', [Validators.email], [this.emailAvailabilityValidator()]]
// Note: async validators go in third array

❌ Common Mistakes

1. Forgetting to Use formControlName on Inputs

<!-- ❌ Wrong - input won't be tied to form control -->
<input [(ngModel)]="email">

<!-- ✅ Correct -->
<input formControlName="email">

2. Mixing ngModel with Reactive Forms

<!-- ❌ Wrong - don't mix these approaches -->
<input formControlName="email" [(ngModel)]="email">

<!-- ✅ Correct - use only formControlName -->
<input formControlName="email">

3. Not Casting FormArray Controls

<!-- ❌ Wrong - TypeScript won't know the type -->
get warehouses() { return this.form.get('warehouses'); }

<!-- ✅ Correct - cast to FormArray -->
get warehouses() { return this.form.get('warehouses') as FormArray; }

4. Showing Errors on All Invalid Fields

Shows errors immediately before user can type, frustrating experience

<!-- ❌ Wrong - shows errors too early -->
<span *ngIf="email.invalid">Invalid email</span>

<!-- ✅ Correct - wait for user to interact -->
<span *ngIf="email.invalid && (email.dirty || email.touched)">Invalid</span>

🎓 Interview Q&A

Q1: What's the difference between Template-Driven and Reactive Forms?

Answer:

  • Template-Driven: Logic in HTML, uses ngModel, easier for simple forms, harder to test
  • Reactive: Logic in TypeScript, uses FormControl, better for complex forms, easier to test
  • Rule of thumb: Use Reactive for complex validation, Template-Driven for simple CRUD forms

Q2: How do you validate that two password fields match?

Answer: Use a group-level custom validator on the FormGroup, not on individual controls:

function passwordMatchValidator(): ValidatorFn {
  return (formGroup: AbstractControl): ValidationErrors | null => {
    const password = formGroup.get('password');
    const confirmPassword = formGroup.get('confirmPassword');

    return password?.value === confirmPassword?.value ? null : { passwordMismatch: true };
  };
}

// In component:
this.form = this.fb.group({
  password: ['', Validators.required],
  confirmPassword: ['', Validators.required]
}, { validators: passwordMatchValidator() });

Q3: How do you implement async validation (e.g., check if email exists)?

Answer: Pass an async validator function as the third parameter:

email: ['', [Validators.email], [this.emailExistsValidator()]],

// Async validator returns Observable/Promise of errors
private emailExistsValidator(): AsyncValidatorFn {
  return (control: AbstractControl): Observable<ValidationErrors | null> => {
    return this.userService.checkEmailExists(control.value).pipe(
      map(exists => exists ? { emailTaken: true } : null),
      debounceTime(300),
      first()
    );
  };
}
