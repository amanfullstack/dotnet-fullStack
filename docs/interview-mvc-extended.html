🖥️ ASP.NET MVC Interview Questions

15 essential interview Q&As covering MVC architecture, controllers, views, models, and real examples from ProductCatalogMVC project.

💡

What's tested in interviews? MVC pattern, Controllers & Actions, Views & Razor, Model Binding, Dependency Injection, View Components, State Management (ViewData, TempData, ViewBag), and Real-world patterns.

Q1: What is the MVC Pattern and why use it?

Click to expand answer

Answer: MVC (Model-View-Controller) separates your application into three layers:

  • Model: Data and business logic (Product, ProductService)
  • View: User interface (Razor .cshtml files)
  • Controller: Handles user requests and orchestrates M & V (ProductsController)

Why? Separation of concerns, easier testing, reusable components, and maintainable code.

Real Example:

// Controllers/ProductsController.cs
public class ProductsController : Controller
{
    private readonly IProductService _productService;

    public ProductsController(IProductService productService)
    {
        _productService = productService; // DI
    }

    public async Task<IActionResult> Index()
    {
        // Fetch from Model (Service)
        var products = await _productService.GetAllProductsAsync();
        // Pass to View
        return View(products);
    }
}

Q2: Explain Controllers and Action Methods in ASP.NET MVC

Click to expand answer

Answer: A Controller is a class that handles HTTP requests. Action Methods are public methods in a Controller that respond to HTTP requests.

HTTP Methods Mapping:

  • GET: Retrieve data (display page)
  • POST: Submit data (form submission)
  • PUT: Update entire resource
  • DELETE: Remove resource
  • PATCH: Partial update

Code Example:

// CRUD Actions from ProductsController
[HttpGet]
public async Task<IActionResult> Index(string search, string category)
{
    var products = await _productService.GetAllProductsAsync();
    if (!string.IsNullOrEmpty(search))
        products = products.Where(p => p.Name.Contains(search)).ToList();
    return View(products);
}

[HttpGet]
public IActionResult Create()
{
    return View(); // Display form
}

[HttpPost]
public async Task<IActionResult> Create(Product product)
{
    if (!ModelState.IsValid)
        return View(product);

    await _productService.CreateProductAsync(product);
    TempData["Success"] = "Product created!";
    return RedirectToAction("Index");
}

Q3: What are Views and Razor? How do you pass data to views?

Click to expand answer

Answer: Views are .cshtml files containing HTML + Razor syntax. Razor lets you mix C# with HTML.

Ways to pass data to views:

  • Model: View(dataObject)
  • ViewBag: Dynamic, not strongly-typed
  • ViewData: Dictionary-based
  • TempData: One-time data (survives redirect)

Razor Example (Views/Products/Index.cshtml):

@model List<Product>

<h2>Product List</h2>
<table class="table">
  <thead>
    <tr>
      <th>Name</th>
      <th>Price</th>
      <th>Actions</th>
    </tr>
  </thead>
  <tbody>
    @foreach (var product in Model)
    {
      <tr>
        <td>@product.Name</td>
        <td>$@product.Price.ToString("F2")</td>
        <td>
          <a href="@Url.Action("Edit", new { id = product.Id })">Edit</a>
        </td>
      </tr>
    }
  </tbody>
</table>

Q4: Explain Model Binding in ASP.NET MVC

Click to expand answer

Answer: Model Binding automatically converts HTTP request data (form, query, route) into C# objects.

// From form data to Product model
[HttpPost]
public async Task<IActionResult> Create(Product product)
{
    // ASP.NET automatically maps form fields to Product properties
    // Form field "Name" → product.Name
    // Form field "Price" → product.Price

    if (!ModelState.IsValid) // Validates using [Required], [Range], etc.
        return View(product);

    await _productService.CreateProductAsync(product);
    return RedirectToAction("Index");
}

Sources of Model Binding: Form data, Query string, Route values, Custom sources

Q5: What is Dependency Injection in ASP.NET MVC and why is it important?

Click to expand answer

Answer: DI is a pattern that provides dependencies (services) to a class instead of the class creating them. Makes code testable and loosely coupled.

// Program.cs - Register services
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddScoped<IProductService, ProductService>();
builder.Services.AddHttpClient();

// ProductsController - Inject IProductService
public class ProductsController : Controller
{
    private readonly IProductService _productService;

    // Constructor Injection
    public ProductsController(IProductService productService)
    {
        _productService = productService;
    }
}

Benefits: Loose coupling, testability, flexibility, centralized configuration

Q6: What is TempData and how is it different from ViewData?

Click to expand answer

ViewData: Dictionary for passing data from Controller to View (same request)

TempData: Dictionary that survives redirects (one-time use)

// ProductsController.cs - Setting success message
[HttpPost]
public async Task<IActionResult> Create(Product product)
{
    if (!ModelState.IsValid)
        return View(product);

    await _productService.CreateProductAsync(product);

    // Message persists through redirect
    TempData["Success"] = "Product created successfully!";
    return RedirectToAction("Index");
}

// In View (Views/Products/Index.cshtml)
@if (TempData["Success"] != null)
{
    <div class="alert alert-success">
        @TempData["Success"]
    </div>
}

Q7: Explain the Request-Response Cycle in ASP.NET MVC

Click to expand answer

Step-by-step:

  1. User enters URL or submits form (HTTP Request)
  2. Routing matches URL to Controller/Action
  3. Controller selected and Action method invoked
  4. Model Binding converts request data to objects
  5. Action executes business logic (CRUD via Service)
  6. Result returned (View, Redirect, Json)
  7. View Engine renders Razor template + data
  8. HTML sent back to browser (HTTP Response)

Example Flow for "Create Product":

Form Submit → ProductsController.Create(POST) → ProductService.CreateProductAsync() → TempData Message → RedirectToAction("Index") → ProductsController.Index(GET) → Service.GetAllProductsAsync() → View rendered with success message

Q8: What is Routing in ASP.NET MVC?

Click to expand answer

Answer: Routing maps URLs to Controllers and Actions.

// Program.cs - Convention-based routing
app.MapDefaultControllerRoute();
// Pattern: /{controller}/{action}/{id?}

// Examples:
// /products/index → ProductsController.Index()
// /products/create → ProductsController.Create()
// /products/edit/5 → ProductsController.Edit(5)

// Or use Attribute routing:
[Route("products")]
public class ProductsController : Controller
{
    [HttpGet("")]
    public async Task<IActionResult> Index() { }

    [HttpGet("{id}")]
    public async Task<IActionResult> Details(int id) { }
}

Q10: What is the Service Layer pattern and why use it?

Click to expand answer

Answer: Service Layer abstracts business logic and data access. Controllers use services instead of directly accessing databases.

// Services/IProductService.cs
public interface IProductService
{
    Task<List<Product>> GetAllProductsAsync();
    Task<Product> GetProductByIdAsync(int id);
    Task CreateProductAsync(Product product);
    Task UpdateProductAsync(Product product);
    Task DeleteProductAsync(int id);
}

// Services/ProductService.cs - Implementation
public class ProductService : IProductService
{
    private readonly HttpClient _httpClient;

    public ProductService(HttpClient httpClient)
    {
        _httpClient = httpClient;
    }

    public async Task<List<Product>> GetAllProductsAsync()
    {
        var response = await _httpClient.GetAsync("http://localhost:5000/api/products");
        var json = await response.Content.ReadAsStringAsync();
        return JsonConvert.DeserializeObject<List<Product>>(json);
    }
}

// In Controller - Use service
public class ProductsController : Controller
{
    private readonly IProductService _productService;

    public ProductsController(IProductService productService)
    {
        _productService = productService;
    }

    public async Task<IActionResult> Index()
    {
        var products = await _productService.GetAllProductsAsync();
        return View(products);
    }
}

Benefits: Testability, separation of concerns, code reuse

Q11: How does HttpClient work in MVC for API integration?

Click to expand answer 
// Program.cs - Register HttpClient
builder.Services.AddHttpClient();

// ProductService.cs - Using HttpClient to call Web API
public class ProductService : IProductService
{
    private readonly HttpClient _httpClient;

    public ProductService(HttpClient httpClient)
    {
        _httpClient = httpClient;
        _httpClient.BaseAddress = new Uri("http://localhost:5000/api/");
    }

    public async Task<List<Product>> GetAllProductsAsync()
    {
        try
        {
            var response = await _httpClient.GetAsync("products");
            response.EnsureSuccessStatusCode();
            var json = await response.Content.ReadAsStringAsync();
            return JsonConvert.DeserializeObject<List<Product>>(json) ?? new();
        }
        catch (Exception ex)
        {
            // Handle error
            return new();
        }
    }

    public async Task CreateProductAsync(Product product)
    {
        var json = JsonConvert.SerializeObject(product);
        var content = new StringContent(json, Encoding.UTF8, "application/json");
        var response = await _httpClient.PostAsync("products", content);
        response.EnsureSuccessStatusCode();
    }
}

Q12: How do you handle errors and exceptions in MVC?

Click to expand answer

Answer: Use try-catch in services, middleware for global errors, and display messages in views.

// Program.cs - Global exception middleware
app.UseExceptionHandler("/Home/Error");

// Error Handler Action
public IActionResult Error()
{
    var exceptionDetails = Activity.Current?.Id ?? HttpContext.TraceIdentifier;
    return View(new ErrorViewModel { RequestId = exceptionDetails });
}

// In Service
public async Task CreateProductAsync(Product product)
{
    try
    {
        var json = JsonConvert.SerializeObject(product);
        var content = new StringContent(json, Encoding.UTF8, "application/json");
        var response = await _httpClient.PostAsync("products", content);
        response.EnsureSuccessStatusCode();
    }
    catch (HttpRequestException ex)
    {
        // Log and re-throw or return default
        throw new InvalidOperationException("Failed to create product", ex);
    }
}

// In Views - Display error messages
@if (ViewData["Error"] != null)
{
    <div class="alert alert-danger">@ViewData["Error"]</div>
}
