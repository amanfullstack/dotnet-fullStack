🗺️ Router & Navigation

Build multi-page applications with routing, lazy loading, and route guards.

What Is Angular Router?

The Angular Router enables navigation between different components based on the URL. It turns your single-page app into a multi-page experience where users can bookmark URLs, use back/forward buttons, and navigate without full page reloads.

Think of it like a sitemap for your Angular app - defining which component to show for each URL path.

Why Router Matters

  • Bookmarkable URLs: Users can save and share links to specific app pages
  • Browser Navigation: Back/forward buttons work naturally
  • Multi-Page Feel: Create apps that feel like traditional websites
  • Lazy Loading: Load feature modules only when needed (performance)
  • Route Guards: Protect pages behind authentication/authorization
  • URL Parameters: Pass data through the URL (e.g., /products/123)

Key Concepts

🛣️ Routes Configuration

Define all app routes in one place:

const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'products', component: ProductListComponent },
  { path: 'products/:id', component: ProductDetailComponent },
  { path: '', redirectTo: '/home', pathMatch: 'full' },
  { path: '**', component: PageNotFoundComponent } // Catch-all
];

🔗 RouterLink & RouterOutlet

  • routerLink="/path" - Navigate without page reload
  • <router-outlet> - Container for routed components
  • routerLinkActive="active" - Highlight current link
<a routerLink="/products" routerLinkActive="active">
  Products
</a>

<!-- Routed component renders here -->
<router-outlet></router-outlet>

📍 Route Parameters

Capture dynamic values from URL:

  • Route params: /products/123 (required, part of path)
  • Query params: /products?sort=name (optional, after ?)

🔐 Route Guards

Control who can access routes:

  • CanActivate - Can user enter this route?
  • CanDeactivate - Can user leave without saving?
  • Resolve - Load data before showing component

⚡ Lazy Loading

Load feature modules only when user navigates to them:

{
  path: 'admin',
  loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule)
}

Code Examples

Example 1: Basic Routing Setup

// app-routing.module.ts
import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './pages/home/home.component';
import { ProductListComponent } from './pages/products/product-list.component';
import { ProductDetailComponent } from './pages/products/product-detail.component';
import { PageNotFoundComponent } from './pages/page-not-found.component';

const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'products', component: ProductListComponent },
  { path: 'products/:id', component: ProductDetailComponent },

  // Default route
  { path: '', redirectTo: '/home', pathMatch: 'full' },

  // Catch-all for 404
  { path: '**', component: PageNotFoundComponent }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

<!-- app.component.html -->
<nav>
  <!-- Navigation links -->
  <a routerLink="/home" routerLinkActive="active">Home</a>
  <a routerLink="/products" routerLinkActive="active">Products</a>
</nav>

<!-- Components render here based on current route -->
<router-outlet></router-outlet>

Example 2: Route Parameters and Query Params

// product-detail.component.ts
import { Component, OnInit } from '@angular/core';
import { ActivatedRoute } from '@angular/router';

@Component({
  selector: 'app-product-detail',
  templateUrl: './product-detail.component.html'
})
export class ProductDetailComponent implements OnInit {
  productId: number;
  sortBy: string;

  constructor(private route: ActivatedRoute) {}

  ngOnInit() {
    // Read route parameter (e.g., /products/123)
    this.route.params.subscribe(params => {
      this.productId = params['id'];
      console.log('Product ID:', this.productId);
      this.loadProduct();
    });

    // Read query parameters (e.g., /products/123?sort=price)
    this.route.queryParams.subscribe(queryParams => {
      this.sortBy = queryParams['sort'] || 'name';
      console.log('Sort by:', this.sortBy);
    });
  }

  loadProduct() {
    // Fetch product based on ID
    console.log('Loading product:', this.productId);
  }
}

<!-- product-detail.component.html -->
<h2>Product #{{productId}}</h2>
<p>Sorted by: {{sortBy}}</p>

<!-- Navigate with route params -->
<a [routerLink]="['/products', 456]">View Product 456</a>

<!-- Navigate with query params -->
<a [routerLink]="['/products', productId]" [queryParams]="{sort: 'price'}">
  Sort by Price
</a>

Example 3: Route Guards (Authentication)

// auth.guard.ts
import { Injectable } from '@angular/core';
import { CanActivate, Router } from '@angular/router';
import { AuthService } from './auth.service';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {
  constructor(private authService: AuthService, private router: Router) {}

  canActivate(): boolean {
    if (this.authService.isLoggedIn()) {
      return true; // Allow navigation
    } else {
      // Redirect to login if not authenticated
      this.router.navigate(['/login']);
      return false;
    }
  }
}

// app-routing.module.ts
const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'login', component: LoginComponent },
  {
    path: 'admin',
    component: AdminComponent,
    canActivate: [AuthGuard] // Protect this route
  }
];

Example 4: Lazy Loading Feature Modules

// app-routing.module.ts
const routes: Routes = [
  { path: 'home', component: HomeComponent },
  { path: 'products', component: ProductListComponent },

  // Lazy load admin module - only loads when user navigates to /admin
  {
    path: 'admin',
    loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule)
  },

  // Lazy load reports module - only loads when user navigates to /reports
  {
    path: 'reports',
    loadChildren: () => import('./reports/reports.module').then(m => m.ReportsModule)
  }
];

// admin/admin-routing.module.ts
const adminRoutes: Routes = [
  { path: '', component: AdminDashboardComponent },
  { path: 'users', component: UserManagementComponent },
  { path: 'settings', component: SettingsComponent }
];

// Now available at /admin, /admin/users, /admin/settings

Example 5: Nested Routes

const routes: Routes = [
  {
    path: 'products',
    component: ProductsLayoutComponent, // Parent layout
    children: [
      { path: '', component: ProductListComponent },
      { path: ':id', component: ProductDetailComponent },
      { path: ':id/reviews', component: ProductReviewsComponent }
    ]
  }
];

// Now you can navigate to:
// /products - Shows product list
// /products/123 - Shows product 123 details
// /products/123/reviews - Shows reviews for product 123

✅ Best Practices

1. Use routerLink Instead of window.location

Why: routerLink navigates without page reload, preserving app state

❌ Bad: <a href="/products"> or window.location.href

✅ Good: <a routerLink="/products">

2. Use Lazy Loading for Feature Modules

Why: Reduces initial bundle size, faster app startup

Rule: If a feature isn't needed on first load, lazy load it

3. Always Unsubscribe from Route Params

Why: Prevents memory leaks when component is destroyed

subscription = this.route.params.subscribe(...);

ngOnDestroy() {
  this.subscription.unsubscribe();
}

4. Use Route Guards for Protected Routes

Why: Enforce authentication before showing pages

{
  path: 'admin',
  component: AdminComponent,
  canActivate: [AuthGuard]
}

5. Handle 404 with Wildcard Route

Why: Shows friendly error page instead of blank screen

// Always last in routes array
{ path: '**', component: PageNotFoundComponent }

❌ Common Mistakes

1. Missing <router-outlet>

Routed components won't display without the outlet container

<!-- ✅ Correct - don't forget this -->
<router-outlet></router-outlet>

2. Route Order Matters

<!-- ❌ Wrong - wildcard catches everything -->
{ path: '**', component: NotFoundComponent },
{ path: 'admin', component: AdminComponent } // Never reached!

<!-- ✅ Correct - specific routes first -->
{ path: 'admin', component: AdminComponent },
{ path: '**', component: NotFoundComponent }

3. Not Using pathMatch: 'full'

<!-- ❌ Wrong - might match /home-settings too -->
{ path: 'home', redirectTo: '/dashboard' }

<!-- ✅ Correct - only matches exact path -->
{ path: 'home', redirectTo: '/dashboard', pathMatch: 'full' }

4. Forgetting to Export RouterModule

Child components won't have access to routing directives

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule] // Don't forget this!
})

🎓 Interview Q&A

Q1: What's the difference between route params and query params?

Answer:

  • Route Params: Part of the URL path, required - /products/123
  • Query Params: After the ? in URL, optional - /products?sort=price&page=2
  • Use route params for essential data (product ID)
  • Use query params for optional filters (sorting, pagination)

Q2: How do you implement route guards and why would you use them?

Answer: Route guards prevent unauthorized access using CanActivate, CanDeactivate, or Resolve:

// CanActivate - Prevent entering a route
@Injectable()
export class AuthGuard implements CanActivate {
  canActivate(route: ActivatedRouteSnapshot): boolean {
    return this.authService.isLoggedIn() || this.router.navigate(['/login']);
  }
}

// Use cases:
// - Block admin pages unless user is admin
// - Redirect to login if not authenticated
// - Load required data before showing component
// - Prevent navigation if form has unsaved changes

Q3: What's the benefit of lazy loading and how do you implement it?

Answer: Lazy loading loads feature modules only when user navigates to them, reducing initial bundle size and improving performance:

{
  path: 'admin',
  loadChildren: () => import('./admin/admin.module').then(m => m.AdminModule)
}

// Benefits:
// - Smaller initial bundle (faster first load)
// - Better for large apps with many features
// - Each feature loads independently

// Child module has its own routes:
// admin-routing.module.ts
const routes = [
  { path: '', component: AdminDashboard },
  { path: 'users', component: UserMgmt }
]
// Now accessible at /admin and /admin/users
