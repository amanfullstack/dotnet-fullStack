💼 Services & HttpClient

Build service layers and consume APIs using HttpClient in MVC applications.

What Are Services & HttpClient?

Services are classes that contain business logic and handle operations like database access, API calls, or complex calculations. They're reusable, testable, and keep logic out of controllers.

HttpClient is .NET's tool for making HTTP requests to external APIs. Use it to consume REST APIs from other services, gather data, and integrate with third-party systems.

Why Services Matter

  • Separation of concerns: Business logic stays in services, not controllers
  • Reusability: Use the same service across multiple controllers
  • Testability: Easy to mock services for unit tests
  • Maintainability: Changes to business logic in one place
  • External integration: HttpClient lets you call other APIs seamlessly

Key Concepts

🏢 Service Layer Pattern

Services sit between controllers and data access. Clean architecture:

  • Controller - Handles HTTP requests, calls services
  • Service - Business logic, calls repositories
  • Repository - Data access, talks to database
  • Database - Persists data

📡 HttpClient Basics

Making HTTP requests to APIs:

// GET - Retrieve data
var response = await _httpClient.GetAsync("https://api.example.com/users");

// POST - Send data
var content = new StringContent(json, Encoding.UTF8, "application/json");
var response = await _httpClient.PostAsync(url, content);

// PUT - Update entire resource
await _httpClient.PutAsync(url, content);

// DELETE - Remove resource
await _httpClient.DeleteAsync(url);

🔄 Async/Await with HttpClient

HttpClient operations are async. Always use async/await:

  • Don't block threads waiting for network responses
  • Improves server scalability dramatically
  • Use Task<T> return type for async methods
  • Let the caller decide to await or not

🛡️ Error Handling in Services

Network calls can fail. Always handle exceptions:

  • HttpRequestException - Network/DNS error
  • TaskCanceledException - Request timeout
  • JsonException - JSON deserialization failed
  • Log errors, return user-friendly messages

Code Examples

Example 1: Service with Repository Pattern

// Interface for dependency injection
public interface IProductService
{
    Task<IEnumerable<ProductViewModel>> GetAllProductsAsync();
    Task<ProductViewModel> GetProductByIdAsync(int id);
    Task<ProductViewModel> CreateProductAsync(ProductViewModel model);
    Task UpdateProductAsync(int id, ProductViewModel model);
    Task DeleteProductAsync(int id);
}

// Implementation
public class ProductService : IProductService
{
    private readonly IProductRepository _repository;
    private readonly ILogger<ProductService> _logger;

    public ProductService(IProductRepository repository, ILogger<ProductService> logger)
    {
        _repository = repository;
        _logger = logger;
    }

    public async Task<IEnumerable<ProductViewModel>> GetAllProductsAsync()
    {
        try
        {
            var products = await _repository.GetAllAsync();
            return products.Select(p => new ProductViewModel
            {
                Id = p.Id,
                Name = p.Name,
                Price = p.Price,
                Description = p.Description
            });
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Error fetching products");
            throw;
        }
    }

    public async Task<ProductViewModel> GetProductByIdAsync(int id)
    {
        if (id <= 0)
            throw new ArgumentException("Invalid product ID");

        var product = await _repository.GetByIdAsync(id);

        if (product == null)
            throw new KeyNotFoundException($"Product {id} not found");

        return new ProductViewModel
        {
            Id = product.Id,
            Name = product.Name,
            Price = product.Price,
            Description = product.Description
        };
    }

    public async Task<ProductViewModel> CreateProductAsync(ProductViewModel model)
    {
        if (string.IsNullOrWhiteSpace(model.Name))
            throw new ArgumentException("Product name required");

        var product = new Product
        {
            Name = model.Name,
            Price = model.Price,
            Description = model.Description,
            CreatedDate = DateTime.UtcNow
        };

        await _repository.AddAsync(product);
        await _repository.SaveAsync();

        model.Id = product.Id;
        return model;
    }
}

Example 2: Consuming External API with HttpClient

public interface IExternalWeatherService
{
    Task<WeatherData> GetWeatherAsync(string city);
}

public class ExternalWeatherService : IExternalWeatherService
{
    private readonly HttpClient _httpClient;
    private readonly ILogger<ExternalWeatherService> _logger;
    private const string ApiBaseUrl = "https://api.weatherapi.com/v1";
    private readonly string _apiKey;

    public ExternalWeatherService(HttpClient httpClient,
        ILogger<ExternalWeatherService> logger,
        IConfiguration config)
    {
        _httpClient = httpClient;
        _logger = logger;
        _apiKey = config["ExternalApis:WeatherApi:Key"];
    }

    public async Task<WeatherData> GetWeatherAsync(string city)
    {
        if (string.IsNullOrWhiteSpace(city))
            throw new ArgumentException("City name required");

        try
        {
            var url = $"{ApiBaseUrl}/current.json?key={_apiKey}&q={Uri.EscapeDataString(city)}";

            // GET request with timeout
            using (var cts = new CancellationTokenSource(TimeSpan.FromSeconds(10)))
            {
                var response = await _httpClient.GetAsync(url, cts.Token);

                if (!response.IsSuccessStatusCode)
                {
                    _logger.LogWarning($"Weather API returned {response.StatusCode}");
                    return null;
                }

                var json = await response.Content.ReadAsStringAsync();
                var weatherData = JsonSerializer.Deserialize<WeatherData>(json);
                return weatherData;
            }
        }
        catch (TaskCanceledException ex)
        {
            _logger.LogError(ex, "Weather API request timeout");
            throw new Exception("Request to weather service timed out");
        }
        catch (HttpRequestException ex)
        {
            _logger.LogError(ex, "Error calling weather API");
            throw new Exception("Failed to fetch weather data");
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "Unexpected error in weather service");
            throw;
        }
    }
}

// DTO for external API response
public class WeatherData
{
    public Location Location { get; set; }
    public Current Current { get; set; }
}

public class Location
{
    public string Name { get; set; }
    public string Country { get; set; }
}

public class Current
{
    public decimal TempC { get; set; }
    public string Condition { get; set; }
}

Example 3: Registering Services in Startup

// Program.cs
var builder = WebApplication.CreateBuilder(args);

// Add services
builder.Services.AddControllersWithViews();

// Register custom services
builder.Services.AddScoped<IProductService, ProductService>();
builder.Services.AddScoped<IProductRepository, ProductRepository>();
builder.Services.AddScoped<IExternalWeatherService, ExternalWeatherService>();

// Register HttpClient with base URL and timeout
builder.Services.AddHttpClient<IExternalWeatherService, ExternalWeatherService>()
    .ConfigureHttpClient(client =>
    {
        client.BaseAddress = new Uri("https://api.weatherapi.com");
        client.Timeout = TimeSpan.FromSeconds(30);
    });

// Add logging
builder.Services.AddLogging();

var app = builder.Build();
app.UseRouting();
app.MapControllerRoute("default", "{controller=Home}/{action=Index}/{id?}");
app.Run();

✅ Best Practices

1. Always Use Dependency Injection

Why: Makes services testable, loosely coupled, easy to swap implementations

❌ Bad:

public class ProductController : Controller
{
    public IActionResult Index()
    {
        var service = new ProductService(); // Hard to test
        var products = service.GetAll();
        return View(products);
    }
}

✅ Good:

public class ProductController : Controller
{
    private readonly IProductService _service;

    public ProductController(IProductService service)
    {
        _service = service; // Injected
    }

    public async Task<IActionResult> Index()
    {
        var products = await _service.GetAllAsync();
        return View(products);
    }
}

2. Always Use Async/Await

Why: Prevents thread blocking, improves scalability

❌ Bad:

public List<Product> GetProducts()
{
    var data = _httpClient.GetStringAsync(url).Result; // Blocks thread!
    return JsonConvert.DeserializeObject<List<Product>>(data);
}

✅ Good:

public async Task<List<Product>> GetProductsAsync()
{
    var data = await _httpClient.GetStringAsync(url); // Non-blocking
    return JsonConvert.DeserializeObject<List<Product>>(data);
}

3. Handle HttpClient Exceptions

Why: Network calls can fail. Provide meaningful error messages

try
{
    var response = await _httpClient.GetAsync(url);

    if (!response.IsSuccessStatusCode)
    {
        _logger.LogWarning($"API returned {response.StatusCode}");
        // Handle 404, 500, etc.
    }

    var content = await response.Content.ReadAsStringAsync();
    return JsonConvert.DeserializeObject<T>(content);
}
catch (TaskCanceledException)
{
    _logger.LogError("Request timeout");
    throw new Exception("External service is not responding");
}
catch (HttpRequestException ex)
{
    _logger.LogError(ex, "Network error");
    throw new Exception("Unable to connect to external service");
}

4. Use TimeSpans for Timeouts

Why: Prevents hanging indefinitely on slow networks

using (var cts = new CancellationTokenSource(TimeSpan.FromSeconds(10)))
{
    var response = await _httpClient.GetAsync(url, cts.Token);
}

// Or configure globally
builder.Services.AddHttpClient<IMyService, MyService>()
    .ConfigureHttpClient(client => client.Timeout = TimeSpan.FromSeconds(30));

5. Map to DTOs, Not Entities

Why: Insulates your app from external API changes

// Don't deserialize directly to entity
// var product = JsonConvert.DeserializeObject<Product>(json);

// Instead, use a DTO
public class ExternalProductDto
{
    public int Id { get; set; }
    public string Name { get; set; }
}

// Then map to your entity
var dto = JsonConvert.DeserializeObject<ExternalProductDto>(json);
var product = new Product { Id = dto.Id, Name = dto.Name };

🎓 Interview Q&A

Q1: Why use services instead of putting logic in controllers?

Answer:

  • Testability: Mock services for unit tests, test business logic independently
  • Reusability: Multiple controllers can use the same service
  • Maintainability: Change business logic once, applies everywhere
  • Single Responsibility: Controller handles HTTP, service handles logic

Controllers are for HTTP concerns (routing, status codes). Services are for business logic (calculations, database operations).

Q2: What's the difference between HttpClient.Result vs async/await?

Answer:

  • .Result: Synchronous, blocks the current thread until the task completes. Can cause deadlocks and wastes thread pool threads.
  • async/await: Asynchronous, thread is freed while waiting. Other requests can use that thread. Much better scalability.
// Bad - blocks thread
var result = _httpClient.GetStringAsync(url).Result;

// Good - frees thread
var result = await _httpClient.GetStringAsync(url);

Impact: With 100 threads and .Result, you can handle ~100 concurrent users. With async/await, you can handle 1000+.

Q3: How do you test a service that uses HttpClient?

Answer: Mock the HttpClient using dependency injection:

// Unit test
[Fact]
public async Task GetProductAsync_Returns_Product()
{
    // Arrange - Create mock
    var mockHttpClient = new Mock<HttpClient>();
    var mockResponse = new HttpResponseMessage
    {
        StatusCode = System.Net.HttpStatusCode.OK,
        Content = new StringContent(JsonConvert.SerializeObject(new { id = 1, name = "Test" }))
    };

    mockHttpClient
        .Setup(x => x.GetAsync(It.IsAny<string>(), It.IsAny<CancellationToken>()))
        .ReturnsAsync(mockResponse);

    var service = new ProductService(mockHttpClient.Object);

    // Act
    var result = await service.GetProductAsync(1);

    // Assert
    Assert.NotNull(result);
    Assert.Equal("Test", result.Name);
}
