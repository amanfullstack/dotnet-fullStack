    🏗️ Architecture & Project Setup

    How to structure an ASP.NET Core Web API project using N-Tier architecture — Controllers, Services, DbContext, and Program.cs.

    Beginner Web API
    ℹ️

    Why Architecture Matters

    A clean architecture separates concerns so each layer has one job. This makes code easier to test, maintain, and extend. The standard ASP.NET Core pattern uses Controllers for routing, Services for business logic, and a DbContext for data access.

    Project Structure

    A well-organized Web API project follows a layered folder structure. Each folder has a single responsibility.

    Project Structure — ProductCatalogAPI-WebAPI-EFCore 
    ProductCatalogAPI-WebAPI-EFCore/
├── Controllers/
│   ├── ProductsController.cs       ← EF Core REST API (7 endpoints)
│   └── AdoProductsController.cs    ← ADO.NET REST API (same 7 endpoints, raw SQL)
├── Services/
│   ├── IProductService.cs          ← EF Core service interface
│   ├── ProductService.cs           ← EF Core + IMemoryCache implementation
│   └── ProductAdoService.cs        ← ADO.NET (SqlConnection, SqlDataReader)
├── Data/
│   └── ProductDbContext.cs         ← EF Core DbContext + soft-delete filter
├── GraphQL/
│   └── ProductQuery.cs             ← HotChocolate queries, mutations & input types
├── Middleware/
│   ├── GlobalErrorHandlingMiddleware.cs  ← catches all unhandled exceptions
│   ├── RequestLoggingMiddleware.cs       ← logs method, path, status, duration
│   └── PerformanceMiddleware.cs          ← warns on slow requests (> 500ms)
├── Models/
│   └── Product.cs                  ← Entity + DTOs (CreateProductRequest etc.)
├── Migrations/                     ← Auto-generated by: dotnet ef migrations add
├── Properties/
│   └── launchSettings.json
├── Program.cs                      ← DI registration + middleware pipeline
├── appsettings.json                ← Connection string, logging config
└── ProductCatalogAPI.csproj        ← NuGet packages (EF Core, HotChocolate)

    N-Tier Architecture — 3 Data Access Approaches

    The API implements the same 7 endpoints three ways — EF Core (ORM), ADO.NET (raw SQL), and GraphQL (query language) — so you can compare all approaches side by side.

    EF Core pathHTTP → ProductsController → IProductService → ProductService → ProductDbContext → SQL Server
    ADO.NET pathHTTP → AdoProductsController → IProductAdoService → ProductAdoService → SqlConnection → SQL Server
    GraphQL pathPOST /graphql → HotChocolate → ProductQuery / ProductMutation → IProductService → SQL Server
    Request Flows 
    ── EF Core REST ──────────────────────────────────────────────────────
GET /api/products → ProductsController → ProductService → DbContext → DB

── ADO.NET REST ──────────────────────────────────────────────────────
GET /api/ado/products → AdoProductsController → ProductAdoService → SqlConnection → DB

── GraphQL ───────────────────────────────────────────────────────────
POST /graphql  { products { id name price } }
              → HotChocolate → ProductQuery → IProductService → DB

    Program.cs — Real Bootstrap Code

    This is the actual Program.cs from the project. It registers EF Core, ADO.NET, GraphQL (HotChocolate), and three custom middleware components.

    Program.cs 
    var builder = WebApplication.CreateBuilder(args);

string connectionString = builder.Configuration.GetConnectionString("DefaultConnection")
    ?? "Server=(localdb)\\mssqllocaldb;Database=ProductCatalogDB;Trusted_Connection=true;";

// ── EF Core (ORM approach) ───────────────────────────────────────
builder.Services.AddDbContext<ProductDbContext>(options =>
    options.UseSqlServer(connectionString)
           .EnableSensitiveDataLogging(builder.Environment.IsDevelopment()));

builder.Services.AddMemoryCache();       // IMemoryCache for ProductService
builder.Services.AddResponseCaching();  // HTTP response cache headers

// ── Service registrations ────────────────────────────────────────
builder.Services.AddScoped<IProductService, ProductService>();      // EF Core
builder.Services.AddScoped<IProductAdoService, ProductAdoService>(); // ADO.NET

// ── GraphQL (HotChocolate) ───────────────────────────────────────
builder.Services
    .AddGraphQLServer()
    .AddQueryType<ProductQuery>()
    .AddMutationType<ProductMutation>()
    .AddErrorFilter(error =>
    {
        Console.WriteLine($"[GraphQL Error] {error.Exception?.Message}");
        return error;
    });

builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();
builder.Services.AddCors(options =>
    options.AddPolicy("AllowFrontend", p =>
        p.WithOrigins("http://localhost:3000", "http://localhost:4200")
         .AllowAnyMethod().AllowAnyHeader()));

var app = builder.Build();

// ── Middleware pipeline (ORDER MATTERS) ─────────────────────────
// 1. Custom error handler — catches ALL exceptions downstream
app.UseMiddleware<GlobalErrorHandlingMiddleware>();
// 2. Request logger — logs every request/response
app.UseMiddleware<RequestLoggingMiddleware>();
// 3. Performance tracker — warns when request > 500ms
app.UseMiddleware<PerformanceMiddleware>();
// 4. HTTP response caching
app.UseResponseCaching();
// 5. GraphQL endpoint at /graphql
app.MapGraphQL("/graphql");

if (app.Environment.IsDevelopment()) { app.UseSwagger(); app.UseSwaggerUI(); }
app.UseCors("AllowFrontend");
app.MapControllers();
app.Run();

    DbContext Configuration

    The DbContext is the bridge between your C# code and the database. It holds DbSet<T> properties for each entity and configuration in OnModelCreating.

    Data/ProductDbContext.cs 
    public class ProductDbContext : DbContext
{
    public ProductDbContext(DbContextOptions<ProductDbContext> options)
        : base(options) { }

    public DbSet<Product> Products { get; set; }

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        // Soft delete — only return IsActive=true by default
        modelBuilder.Entity<Product>()
            .HasQueryFilter(p => p.IsActive);

        // Index for performance
        modelBuilder.Entity<Product>()
            .HasIndex(p => p.Category);

        // Decimal precision
        modelBuilder.Entity<Product>()
            .Property(p => p.Price)
            .HasColumnType("decimal(18,2)");
    }
}

    Use appsettings.Development.json or environment variables for real connection strings and API keys. The .gitignore template excludes appsettings.*.json by default.

    ADO.NET — Raw SQL Service

    ProductAdoService performs the same operations as ProductService (EF Core) but uses SqlConnection and SqlCommand directly. Route prefix: /api/ado/products

    Services/ProductAdoService.cs — key patterns 
    public class ProductAdoService : IProductAdoService
{
    private readonly string _connectionString;
    private readonly ILogger<ProductAdoService> _logger;

    public ProductAdoService(IConfiguration configuration, ILogger<ProductAdoService> logger)
    {
        // Connection string injected via IConfiguration — never hardcode it!
        _connectionString = configuration.GetConnectionString("DefaultConnection")
            ?? "Server=(localdb)\\mssqllocaldb;Database=ProductCatalogDB;Trusted_Connection=true;";
        _logger = logger;
    }

    // READ ALL — SqlDataReader maps each column manually to Product
    public async Task<List<Product>> GetAllProductsAsync()
    {
        var products = new List<Product>();
        using (SqlConnection conn = new SqlConnection(_connectionString))
        {
            string sql = @"SELECT Id, Name, Description, Price, StockQuantity, Category,
                                  CreatedDate, UpdatedDate, IsActive
                           FROM Products WHERE IsActive = 1 ORDER BY Name";

            SqlCommand cmd = new SqlCommand(sql, conn);
            await conn.OpenAsync();
            _logger.LogInformation("[ADO.NET] Executing: Get All Products");

            using (SqlDataReader reader = await cmd.ExecuteReaderAsync())
            {
                while (await reader.ReadAsync())
                {
                    products.Add(new Product
                    {
                        Id            = (int)reader["Id"],
                        Name          = (string)reader["Name"],
                        Description   = reader["Description"] != DBNull.Value ? (string)reader["Description"] : "",
                        Price         = (decimal)reader["Price"],
                        StockQuantity = (int)reader["StockQuantity"],
                        Category      = (string)reader["Category"],
                        IsActive      = (bool)reader["IsActive"]
                    });
                }
            }
        }
        return products;
    }

    // CREATE — parameterized INSERT + SCOPE_IDENTITY() to get new Id
    public async Task<Product> CreateProductAsync(Product product)
    {
        using (SqlConnection conn = new SqlConnection(_connectionString))
        {
            // IMPORTANT: NEVER use string concatenation — always use @parameters
            string sql = @"INSERT INTO Products (Name, Description, Price, StockQuantity, Category,
                                                CreatedDate, UpdatedDate, IsActive)
                           VALUES (@Name, @Description, @Price, @StockQuantity, @Category,
                                   @CreatedDate, @UpdatedDate, @IsActive);
                           SELECT SCOPE_IDENTITY();";   // ← get inserted row ID

            SqlCommand cmd = new SqlCommand(sql, conn);
            cmd.Parameters.AddWithValue("@Name",          product.Name);
            cmd.Parameters.AddWithValue("@Description",   product.Description ?? "");
            cmd.Parameters.AddWithValue("@Price",         product.Price);
            cmd.Parameters.AddWithValue("@StockQuantity", product.StockQuantity);
            cmd.Parameters.AddWithValue("@Category",      product.Category);
            cmd.Parameters.AddWithValue("@CreatedDate",   DateTime.UtcNow);
            cmd.Parameters.AddWithValue("@UpdatedDate",   DateTime.UtcNow);
            cmd.Parameters.AddWithValue("@IsActive",      true);

            await conn.OpenAsync();
            var result = await cmd.ExecuteScalarAsync();  // ExecuteScalar = single value
            product.Id = Convert.ToInt32(result);
        }
        return product;
    }

    // SOFT DELETE — UPDATE IsActive=0 instead of DELETE
    public async Task<bool> DeleteProductAsync(int id)
    {
        using (SqlConnection conn = new SqlConnection(_connectionString))
        {
            string sql = @"UPDATE Products SET IsActive = 0, UpdatedDate = @UpdatedDate
                           WHERE Id = @Id AND IsActive = 1; SELECT @@ROWCOUNT;";
            SqlCommand cmd = new SqlCommand(sql, conn);
            cmd.Parameters.AddWithValue("@Id",          id);
            cmd.Parameters.AddWithValue("@UpdatedDate", DateTime.UtcNow);
            await conn.OpenAsync();
            var rowsAffected = await cmd.ExecuteScalarAsync();
            return rowsAffected != null && (int)rowsAffected > 0;
        }
    }
}

    Key classesSqlConnection · SqlCommand · SqlDataReader · ExecuteReaderAsync · ExecuteScalarAsync · ExecuteNonQueryAsync

    GraphQL — HotChocolate

    GraphQL is served at /graphql by HotChocolate. The client sends a POST with a query string specifying exactly which fields it needs — no over-fetching.

    GraphQL/ProductQuery.cs — Query & Mutation types 
    // ── Query Type (read operations) ────────────────────────────
public class ProductQuery
{
    // { products { id name price } }
    public async Task<List<Product>> GetProducts([Service] IProductService service)
    {
        Console.WriteLine("[GraphQL] Query: GetProducts");
        return await service.GetAllProductsAsync();
    }

    // { productById(id: 1) { id name price category } }
    public async Task<Product?> GetProductById(int id, [Service] IProductService service)
        => await service.GetProductByIdAsync(id);

    // { productsByCategory(category: "Electronics") { id name } }
    public async Task<List<Product>> GetProductsByCategory(
        string category, [Service] IProductService service)
        => await service.GetProductsByCategoryAsync(category);

    // { expensiveProducts(minPrice: 500) { id name price } }
    public async Task<List<Product>> GetExpensiveProducts(
        decimal minPrice, [Service] IProductService service)
    {
        var all = await service.GetAllProductsAsync();
        return all.Where(p => p.Price >= minPrice).OrderByDescending(p => p.Price).ToList();
    }

    // { lowStockProducts(threshold: 10) { id name stockQuantity } }
    public async Task<List<Product>> GetLowStockProducts(
        int threshold, [Service] IProductService service)
    {
        var all = await service.GetAllProductsAsync();
        return all.Where(p => p.StockQuantity <= threshold).ToList();
    }
}

// ── Mutation Type (write operations) ────────────────────────
public class ProductMutation
{
    // mutation { createProduct(input: { name: "Laptop" price: 999.99 ... }) { id name } }
    public async Task<Product> CreateProduct(
        CreateProductInput input, [Service] IProductService service)
    {
        var product = new Product
        {
            Name          = input.Name,
            Description   = input.Description ?? "",
            Price         = input.Price,
            StockQuantity = input.StockQuantity,
            Category      = input.Category
        };
        return await service.CreateProductAsync(product);
    }

    // mutation { deleteProduct(id: 1) { success message } }
    public async Task<DeleteProductResponse> DeleteProduct(
        int id, [Service] IProductService service)
    {
        var success = await service.DeleteProductAsync(id);
        return new DeleteProductResponse
        {
            Success = success,
            Message = success ? "Deleted successfully" : "Product not found"
        };
    }
}

// ── Input / Response types ───────────────────────────────────
public class CreateProductInput
{
    public string  Name          { get; set; } = string.Empty;
    public string? Description   { get; set; }
    public decimal Price         { get; set; }
    public int     StockQuantity { get; set; }
    public string  Category      { get; set; } = string.Empty;
}

public class DeleteProductResponse { public bool Success { get; set; } public string Message { get; set; } = ""; }

    Interview Tips

    💡

    Common questions on this topic

    • What is the difference between EF Core and ADO.NET? — EF Core is an ORM: write LINQ, get SQL automatically, has migrations. ADO.NET is raw: you write SQL strings, manually map rows, maximum control and performance.
    • How do you prevent SQL injection in ADO.NET? — Always use parameterized queries via cmd.Parameters.AddWithValue("@param", value). Never concatenate user input into SQL strings.
    • What is GraphQL and when would you use it? — A query language for APIs where the client specifies exactly which fields it needs. Use it when clients have diverse data requirements to avoid over-fetching/under-fetching.
    • What is the difference between AddSingleton, AddScoped, and AddTransient? — Singleton: one instance forever. Scoped: one per HTTP request. Transient: new instance every injection.
    • Why separate controllers from services? — Controllers handle HTTP concerns (routing, status codes, request/response). Services handle business logic. This makes services testable without an HTTP context.
