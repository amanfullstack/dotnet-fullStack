15 essential interview Q&As covering components, services, RxJS, Reactive Forms, routing, and examples from ProductUI-Angular project.

Q1: What is Angular and its core concepts?

Click to expand answer

Answer: Angular is a TypeScript-based frontend framework by Google for building Single Page Applications (SPAs).

Core Concepts:

  • Components: Reusable UI building blocks
  • Services: Business logic, data access (shared)
  • Dependency Injection: Manages service instances
  • Routing: Navigation between views
  • RxJS: Reactive programming with Observables
  • Forms: Template-driven or Reactive

Q2: Explain Components in Angular

Click to expand answer

Answer: A component is a reusable UI unit with class logic, template (HTML), and styles.

// product-list.component.ts
import { Component, OnInit } from '@angular/core';
import { ProductService } from '../services/product.service';

@Component({
  selector: 'app-product-list',
  templateUrl: './product-list.component.html',
  styleUrls: ['./product-list.component.css']
})
export class ProductListComponent implements OnInit {
  products: any[] = [];
  loading = false;

  constructor(private productService: ProductService) {}

  ngOnInit(): void {
    this.loadProducts();
  }

  loadProducts(): void {
    this.loading = true;
    this.productService.getProducts().subscribe({
      next: (data) => {
        this.products = data;
        this.loading = false;
      },
      error: (err) => {
        console.error('Error loading products', err);
        this.loading = false;
      }
    });
  }
}
<!-- product-list.component.html -->
<div class="product-list">
  <div *ngIf="loading" class="spinner">Loading...</div>

  <div *ngFor="let product of products" class="product-card">
    <h3>{{ product.name }}</h3>
    <p>{{ product.description }}</p>
    <p class="price">${{ product.price | number:'1.2-2' }}</p>
  </div>
</div>

Q3: What are Services and how do you use Dependency Injection?

Click to expand answer 
// services/product.service.ts
import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class ProductService {
  private apiUrl = 'http://localhost:5000/api/products';

  constructor(private http: HttpClient) {}

  getProducts(): Observable<any[]> {
    return this.http.get<any[]>(this.apiUrl);
  }

  getProduct(id: number): Observable<any> {
    return this.http.get<any>(`${this.apiUrl}/${id}`);
  }

  createProduct(product: any): Observable<any> {
    return this.http.post<any>(this.apiUrl, product);
  }

  updateProduct(id: number, product: any): Observable<any> {
    return this.http.put<any>(`${this.apiUrl}/${id}`, product);
  }

  deleteProduct(id: number): Observable<void> {
    return this.http.delete<void>(`${this.apiUrl}/${id}`);
  }
}

// In Component - Auto-injected
export class ProductListComponent {
  constructor(private productService: ProductService) {}
  // productService is available
}

Benefits: Loose coupling, testability, single responsibility

Q4: Explain RxJS Observables and subscription

Click to expand answer

Answer: Observable is a stream of data that can be subscribed to. Async.

// Using Observables from HttpClient
this.productService.getProducts().subscribe({
  next: (data) => {
    console.log('Products loaded:', data);
    this.products = data;
  },
  error: (err) => console.error('Error:', err),
  complete: () => console.log('Stream completed')
});

// Or using RxJS operators
this.productService.getProducts()
  .pipe(
    map(products => products.filter(p => p.price > 100)),
    tap(filtered => console.log('Filtered:', filtered)),
    catchError(err => {
      console.error('Error:', err);
      return of([]); // Return empty array on error
    })
  )
  .subscribe(data => this.products = data);

// Unsubscribe to avoid memory leaks
subscription.unsubscribe();

// Or use async pipe (auto unsubscribe)
<div *ngFor="let product of (productService.getProducts() | async)">
  {{ product.name }}
</div>

Q5: What are Reactive Forms and how to use them?

Click to expand answer

Answer: Reactive Forms provide explicit, immutable API for form handling. Programmatic and powerful.

// Component - product-form.component.ts
import { Component } from '@angular/core';
import { FormBuilder, FormGroup, Validators } from '@angular/forms';
import { ProductService } from '../services/product.service';

@Component({
  selector: 'app-product-form',
  templateUrl: './product-form.component.html'
})
export class ProductFormComponent {
  productForm: FormGroup;

  constructor(
    private fb: FormBuilder,
    private productService: ProductService
  ) {
    this.productForm = this.fb.group({
      name: ['', [Validators.required, Validators.minLength(3)]],
      description: [''],
      price: ['', [Validators.required, Validators.min(0)]],
      stockQuantity: [0, Validators.required],
      category: ['', Validators.required]
    });
  }

  onSubmit(): void {
    if (this.productForm.invalid) return;

    this.productService.createProduct(this.productForm.value)
      .subscribe({
        next: () => {
          console.log('Product created');
          this.productForm.reset();
        },
        error: (err) => console.error('Error:', err)
      });
  }
}
<!-- product-form.component.html -->
<form [formGroup]="productForm" (ngSubmit)="onSubmit()">
  <div>
    <label>Name</label>
    <input formControlName="name" />
    <span *ngIf="productForm.get('name')?.hasError('required')">
      Name is required
    </span>
  </div>

  <div>
    <label>Price</label>
    <input type="number" formControlName="price" />
    <span *ngIf="productForm.get('price')?.hasError('min')">
      Price must be >= 0
    </span>
  </div>

  <button [disabled]="productForm.invalid">Create</button>
</form>

    this.router.navigate(['/products', this.productId, 'edit']);

Q7: What is the async pipe and why use it?

Click to expand answer

Answer: The async pipe subscribes to Observables/Promises and auto-unsubscribes when component destroys.

// Component
export class ProductListComponent {
  products$ = this.productService.getProducts();

  constructor(private productService: ProductService) {}
}

// Template - async pipe handles subscription
<div *ngFor="let product of (products$ | async)">
  {{ product.name }}
</div>

// Equivalent to (without async pipe):
<div *ngFor="let product of products">
  {{ product.name }}
</div>

// And in component:
products: any[] = [];
ngOnInit(): void {
  this.productService.getProducts().subscribe(data => {
    this.products = data;
  });
}
// Need to manually unsubscribe in ngOnDestroy

// Benefits: Auto-unsubscribe, OnPush change detection, cleaner code

Q8: Explain Change Detection in Angular

Click to expand answer

Answer: Change Detection determines when Angular updates the view based on data changes.

// Default Strategy: CheckAlways (slow, checks entire tree)
@Component({
  selector: 'app-product',
  template: '...'
})
export class ProductComponent {
  data = 'initial';
}

// OnPush Strategy: Only checks when @Input changes or events fire (fast)
@Component({
  selector: 'app-product',
  template: '...',
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class ProductComponent {
  @Input() product: any;
}

// Using OnPush with Observables
@Component({
  selector: 'app-product-list',
  template: `
    <app-product *ngFor="let p of (products$ | async)"
                  [product]="p"
                  (changeDetection)="OnPush">
    </app-product>
  `,
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class ProductListComponent {
  products$ = this.productService.getProducts();
  constructor(private productService: ProductService) {}
}

Q9: What are RxJS operators? Give examples

Click to expand answer

Answer: Operators transform Observable streams. Common ones:

// map: Transform each value
this.productService.getProducts()
  .pipe(
    map(products => products.map(p => ({...p, price: p.price * 1.1})))
  )
  .subscribe(data => console.log(data));

// filter: Keep only matching values
.pipe(filter(product => product.price > 100))

// switchMap: Switch to new Observable (cancel previous)
this.route.params.pipe(
  switchMap(params => this.productService.getProduct(params['id']))
)

// tap: Side effects without changing data
.pipe(tap(data => console.log('Before filter:', data)))

// catchError: Handle errors
.pipe(
  catchError(err => {
    console.error('Error:', err);
    return of([]); // Return fallback
  })
)

// mergeMap: Handle multiple requests
.pipe(
  mergeMap(products => this.getDetails(products))
)

// debounceTime: Wait before emitting
searchControl.valueChanges.pipe(
  debounceTime(300),
  distinctUntilChanged(),
  switchMap(query => this.search(query))
)

Q10: How to handle HTTP interceptors?

Click to expand answer 
// http.interceptor.ts
import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpRequest, HttpHandler, HttpEvent } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable()
export class AuthInterceptor implements HttpInterceptor {
  constructor(private authService: AuthService) {}

  intercept(req: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
    // Add auth token to all requests
    const token = this.authService.getToken();
    const authReq = req.clone({
      setHeaders: { Authorization: `Bearer ${token}` }
    });

    return next.handle(authReq);
  }
}

// app.module.ts - Register interceptor
import { HttpClientModule, HTTP_INTERCEPTORS } from '@angular/common/http';

@NgModule({
  imports: [HttpClientModule],
  providers: [
    {
      provide: HTTP_INTERCEPTORS,
      useClass: AuthInterceptor,
      multi: true
    }
  ]
})
export class AppModule { }

Q11: What are Lifecycle Hooks?

Click to expand answer 
import { Component, OnInit, OnDestroy } from '@angular/core';
import { Subscription } from 'rxjs';

export class ProductListComponent implements OnInit, OnDestroy {
  products: any[];
  subscription: Subscription;

  constructor(private productService: ProductService) {}

  // Called when component initialized
  ngOnInit(): void {
    this.loadProducts();
  }

  // Called on input property change
  ngOnChanges(changes: SimpleChanges): void {
    console.log('Input changed:', changes);
  }

  // Called after view/template created
  ngAfterViewInit(): void {
    console.log('View initialized');
  }

  // Called before component destroyed
  ngOnDestroy(): void {
    // Cleanup: unsubscribe, clear timers
    if (this.subscription) {
      this.subscription.unsubscribe();
    }
  }

  loadProducts(): void {
    this.subscription = this.productService.getProducts()
      .subscribe(data => this.products = data);
  }
}

Common hooks: OnInit, OnChanges, OnDestroy, AfterViewInit

Q12: How do you share data between components?

// 1. Parent to Child: @Input
@Component({
  selector: 'app-child',
  template: '<p>{{ product.name }}</p>'
})
export class ChildComponent {
  @Input() product: any;
}

// Parent template:
<app-child [product]="currentProduct"></app-child>

// 2. Child to Parent: @Output
@Component({
  selector: 'app-child',
  template: '<button (click)="onSelect()">Select</button>'
})
export class ChildComponent {
  @Output() selected = new EventEmitter<any>();

  onSelect(): void {
    this.selected.emit(this.product);
  }
}

// Parent:
<app-child (selected)="onProductSelected($event)"></app-child>
onProductSelected(product: any): void {
  console.log('Selected:', product);
}

// 3. Via Service (Shared):
@Injectable({ providedIn: 'root' })
export class DataService {
  data$ = new BehaviorSubject<any>(null);
}

// Component A:
this.dataService.data$.next(newValue);

// Component B:
this.dataService.data$.subscribe(value => this.data = value);

Q13: How to implement lazy loading in Angular?

Click to expand answer

Answer: Lazy loading loads feature modules only when needed (improves initial load time).

// app-routing.module.ts
const routes: Routes = [
  { path: '', component: HomeComponent },
  {
    path: 'products',
    loadChildren: () => import('./products/products.module')
      .then(m => m.ProductsModule)

    path: 'admin',

Q14: What are Guards and how to implement them?

Click to expand answer

Answer: Guards control access to routes (protect admin pages, login required, etc.)

// auth.guard.ts
import { Injectable } from '@angular/core';
import { CanActivate, Router } from '@angular/router';

@Injectable({ providedIn: 'root' })
export class AuthGuard implements CanActivate {
  constructor(private authService: AuthService, private router: Router) {}

  canActivate(): boolean {
    if (this.authService.isLoggedIn()) {
      return true;
    }

    this.router.navigate(['/login']);
    return false;
  }
}

// In routing:
const routes: Routes = [
  { path: 'admin', component: AdminComponent, canActivate: [AuthGuard] }
];

Q15: TypeScript - What are Interfaces and generics?

Click to expand answer 
// Interface - Define object structure
interface Product {
  id: number;
  name: string;
  price: number;
  description?: string; // Optional
}

// Generics - Reusable with type variables
interface ApiResponse<T> {
  success: boolean;
  data: T;
  error?: string;
}

// Use:
const productResponse: ApiResponse<Product> = {
  success: true,
  data: { id: 1, name: 'Laptop', price: 999 }
};

// Service with generics:
@Injectable()
export class ApiService {
  get<T>(url: string): Observable<T> {
    return this.http.get<T>(url);
  }
}
