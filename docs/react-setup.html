⚛️ React Setup Guide

Complete step-by-step guide to set up and run the ProductUI-React project locally.

Prerequisites

⚠️

Before starting, ensure you have:

  • ✅ Node.js 16+ and npm 7+ installed
  • ✅ ProductCatalogAPI running on localhost:5000
  • ✅ Git (for cloning the repository)
  • ✅ A modern browser (Chrome, Firefox, Safari, or Edge)

1. Check Prerequisites

# Check Node.js version (should be 16+)
node --version

# Check npm version (should be 7+)
npm --version

2. Clone or Navigate to Project

# Using Git
git clone https://github.com/[USER]/dotnet-fullStack.git
cd dotnet-fullStack/ProductUI-React-JavaScript

# Or if already cloned
cd ProductUI-React-JavaScript

3. Verify API is Running

The React app depends on the ProductCatalogAPI. Make sure it's running:

cd ProductCatalogAPI-WebAPI-EFCore
dotnet run

# Should show: "Now listening on: http://localhost:5000"

4. Install Dependencies

# Install npm packages (takes 1-2 minutes)
npm install

# Verify installation
npm --version
ls node_modules | head

5. Configure Environment

Create or verify .env file in project root:

VITE_API_URL=http://localhost:5000/api

6. Start Development Server

# Start Vite dev server
npm run dev

# Server starts on http://localhost:5173 (or next available port)
# Output shows the exact URL

7. Access the Application

  • Home Page: http://localhost:5173
  • Products List: http://localhost:5173/
  • Create Product: http://localhost:5173/create
  • Edit Product: http://localhost:5173/edit/:id
  • Product Detail: http://localhost:5173/:id

Features

✓

Once running, you can:

  • 📋 View all products in a responsive table
  • 🔍 Search products by name or category
  • ➕ Create new products via modal form
  • ✏️ Edit existing product details
  • 👁️ View individual product details
  • 🗑️ Delete products with confirmation
  • 🌙 Toggle dark/light theme
  • 📱 Full responsive design for mobile/tablet
  • ⚡ Fast HMR (Hot Module Replacement) with Vite
  • 🎯 Modern React hooks for state management

Troubleshooting

Issue: Cannot connect to ProductCatalogAPI

Solution: Ensure the API is running on localhost:5000

# In another terminal window, run:
cd ProductCatalogAPI-WebAPI-EFCore
dotnet run

Issue: CORS Error from API

Solution: Configure Vite proxy in vite.config.js:

export default {
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:5000',
        changeOrigin: true,
        rewrite: path => path.replace(/^\/api/, '/api')
      }
    }
  }
}

Issue: Port 5173 already in use

Solution: Vite will automatically use the next available port

# Check what port was used
npm run dev

# Or manually specify a port
npm run dev -- --port 5174
