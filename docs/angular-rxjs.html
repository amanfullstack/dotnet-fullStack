〰️ RxJS & Observables

Master reactive programming with RxJS observables, operators, and subscriptions.

What Are RxJS & Observables?

RxJS is a library for reactive programming using Observables. An Observable is a stream of data that can be observed over time. Instead of imperative "do this then that", you declare "when X happens, do Y".

Observables are core to Angular for handling async operations (HTTP, events, timers).

Key Concepts

〰️ What is an Observable?

A stream of values that can be observed:

  • Lazy: Nothing happens until you subscribe
  • Cancellable: Unsubscribe to stop listening
  • Data over time: Can emit multiple values
  • Transformable: Use operators to manipulate data

🔄 Operators

Functions that transform observable data:

  • map() - Transform values
  • filter() - Keep only matching values
  • switchMap() - Switch to new observable
  • catchError() - Handle errors
  • debounceTime() - Wait before emitting

🎯 Subject vs BehaviorSubject

  • Subject - New subscribers get future values only
  • BehaviorSubject - New subscribers get latest value immediately
  • Use for: sharing data, event buses

Code Examples

Example 1: Basic Observable & Subscription

import { Observable } from 'rxjs';
import { map, filter } from 'rxjs/operators';

// Create observable
const numbers$ = new Observable(subscriber => {
  subscriber.next(1);
  subscriber.next(2);
  subscriber.next(3);
  subscriber.complete();
});

// Subscribe to get values
numbers$.subscribe(
  value => console.log(value), // 1, 2, 3
  error => console.error(error),
  () => console.log('Complete')
);

Example 2: Using Operators

this.productService.getProducts()
  .pipe(
    // Transform each product
    map(products => products.map(p => ({
      ...p,
      displayPrice: `$${p.price}`
    }))),
    // Keep only expensive items
    filter(products => products.some(p => p.price > 100)),
    // Handle errors
    catchError(error => {
      console.error('Error:', error);
      return of([]); // Return empty array on error
    })
  )
  .subscribe(filteredProducts => {
    this.products = filteredProducts;
  });

Example 3: switchMap for Dependent Observables

import { switchMap } from 'rxjs/operators';

// When category changes, fetch new products
this.categoryId$
  .pipe(
    switchMap(categoryId => {
      if (!categoryId) return of([]);
      return this.service.getProductsByCategory(categoryId);
    })
  )
  .subscribe(products => this.products = products);

✅ Best Practices

1. Always Unsubscribe

Why: Prevents memory leaks

subscription = this.service.getData().subscribe(...);
ngOnDestroy() { this.subscription.unsubscribe(); }

2. Use Async Pipe

Why: Automatically unsubscribes

{{ products$ | async }}

3. Handle Errors with catchError

Why: Observable stops on error, need recovery strategy

.pipe(
  catchError(error => {
    console.error('Error:', error);
    return of(defaultValue);
  })
)

4. Use switchMap for Route Changes

Why: Cancels previous subscription automatically

🎓 Interview Q&A

Q1: What's the difference between cold and hot observables?

Answer:

  • Cold: New data for each subscriber (HTTP requests)
  • Hot: Shared data stream, subscribers get current/future values (mouse events)

Q2: Why use switchMap instead of mergeMap?

Answer: switchMap cancels the previous observable when a new one arrives. mergeMap runs all simultaneously. Use switchMap when you only care about the latest.

Q3: When should you use Subject vs Observable?

Answer: Observable for data flow, Subject for event broadcasting. Subjects let you push values manually.
