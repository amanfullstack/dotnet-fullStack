🎣 Hooks & State

Master useState, useEffect, and state management in functional components.

What Are Hooks?

Hooks are special functions that let you "hook into" React features from functional components. Before hooks, you needed class components to use state and lifecycle methods. Now with hooks, functional components can do everything!

Think of hooks as superpowers for functional components - useState adds state, useEffect adds side effects, and more.

Class Components vs Functional Components

Modern React uses functional components with hooks. Class components are the old way, but understanding both helps for interviews and legacy codebases.

Side-by-Side Comparison: Counter Component

Class Component (Old Way)

import React from 'react';

class CounterClass extends React.Component {
  constructor(props) {
    super(props);
    // Initialize state
    this.state = { count: 0 };
  }

  // Lifecycle method - runs on mount
  componentDidMount() {
    console.log('Component mounted');
  }

  // Lifecycle method - runs after update
  componentDidUpdate() {
    console.log('Component updated, count:', this.state.count);
  }

  // Lifecycle method - runs before unmount
  componentWillUnmount() {
    console.log('Component unmounting');
  }

  render() {
    return (
      <div>
        <h1>Count: {this.state.count}</h1>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          Increment
        </button>
      </div>
    );
  }
}

export default CounterClass;

Functional Component with Hooks (Modern Way)

import React, { useState, useEffect } from 'react';

function CounterFunctional() {
  // Hook: Add state
  const [count, setCount] = useState(0);

  // Hook: Lifecycle methods combined
  useEffect(() => {
    console.log('Component mounted or updated');
    return () => console.log('Component unmounting');
  }, []);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}

export default CounterFunctional;

API Fetch Example: Class vs Functional

Class Component - Multiple Lifecycle Methods

class ProductListClass extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      products: [],
      loading: true,
      error: null
    };
  }

  // Load data when component mounts
  componentDidMount() {
    this.loadProducts();
  }

  // Update when productId prop changes
  componentDidUpdate(prevProps) {
    if (prevProps.productId !== this.props.productId) {
      this.loadProducts();
    }
  }

  loadProducts = async () => {
    try {
      const response = await fetch('http://localhost:5000/api/products');
      const data = await response.json();
      this.setState({ products: data, loading: false });
    } catch (error) {
      this.setState({ error: error.message, loading: false });
    }
  };

  render() {
    if (this.state.loading) return <div>Loading...</div>;
    if (this.state.error) return <div>Error: {this.state.error}</div>;

    return (
      <ul>
        {this.state.products.map(p => (
          <li key={p.id}>{p.name}</li>
        ))}
      </ul>
    );
  }
}

Functional Component - Cleaner with Hooks

function ProductListFunctional({ productId }) {
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // Load products when component mounts or productId changes
    const loadProducts = async () => {
      try {
        setLoading(true);
        const response = await fetch('http://localhost:5000/api/products');
        const data = await response.json();
        setProducts(data);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    loadProducts();
  }, [productId]); // Re-run if productId changes

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {products.map(p => (
        <li key={p.id}>{p.name}</li>
      ))}
    </ul>
  );
}

Feature Class Component Functional Component (Hooks)
Syntax class MyComponent extends React.Component function MyComponent()
State this.state = {}, this.setState() useState() hook
Mount componentDidMount() useEffect(() => {}, [])
Update componentDidUpdate() useEffect(() => {}, [deps])

Form Handling Example: Class vs Functional

Class Component Form

class FormClass extends React.Component {
  constructor(props) {
    super(props);
    this.state = { name: '', email: '' };
  }

  handleChange = (e) => {
    const { name, value } = e.target;
    this.setState({ [name]: value });
  };

  handleSubmit = (e) => {
    e.preventDefault();
    console.log(this.state);
  };

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input
          name="name"
          value={this.state.name}
          onChange={this.handleChange}
        />
        <input
          name="email"
          value={this.state.email}
          onChange={this.handleChange}
        />
        <button type="submit">Submit</button>
      </form>
    );
  }
}

Functional Component Form (Much Simpler!)

function FormFunctional() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log({ name, email });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={name}
        onChange={(e) => setName(e.target.value)}
      />
      <input
        value={email}
        onChange={(e) => setEmail(e.target.value)}
      />
      <button type="submit">Submit</button>
    </form>
  );
}

Why Modern React Uses Functional Components

  1. Simpler Syntax: Just write functions, no class boilerplate
  2. Easier to Test: Pure functions are easier to unit test
  3. Better Performance: Hooks enable better optimization, smaller bundle
  4. Code Organization: Related logic grouped in useEffect, not split across lifecycle methods
  5. Easier to Share Logic: Custom hooks reuse logic between components
  6. No 'this' Keyword: No binding issues, no confusion about context
  7. Smaller Learning Curve: JavaScript functions are more familiar than OOP classes
✓

Best Practice: Always use functional components with hooks for new React code. Class components should only be used in legacy projects or when specifically required.

Why Hooks Matter

  • Simpler Syntax: Functional components are easier to read and write than classes
  • Code Reuse: Share logic between components with custom hooks
  • Better Organization: Group related logic together instead of splitting across lifecycle methods
  • Easier Testing: Test pure functions instead of class instances
  • Smaller Bundle: Functional components compile to less code

Key Concepts

🔄 useState Hook

Adds state to functional components. Returns [currentValue, setterFunction]:

const [count, setCount] = useState(0);
// count = current value (0)
// setCount = function to update count

const [name, setName] = useState('John');
const [items, setItems] = useState([]);
const [user, setUser] = useState({ name: '', age: 0 });

⚡ useEffect Hook

Run side effects like API calls, timers, subscriptions after component renders:

// Run after every render
useEffect(() => {
  console.log('Component rendered');
});

// Run after render only if dependencies changed
useEffect(() => {
  loadData();
}, [userId]); // Re-run if userId changes

// Run once on mount
useEffect(() => {
  console.log('Component mounted');
}, []);

🧹 Cleanup Function

Return a function from useEffect to clean up (unsubscribe, clear timers):

useEffect(() => {
  const timer = setTimeout(() => {
    setCount(c => c + 1);
  }, 1000);

  // Cleanup function - called before component unmounts or effect re-runs
  return () => clearTimeout(timer);
}, []);

🎯 Dependency Array

Controls when useEffect runs:

  • No array: Runs after EVERY render (be careful!)
  • Empty []: Runs ONCE on mount (perfect for setup)
  • [dep1, dep2]: Runs when dependencies change

📏 Rules of Hooks

  • Only call hooks at top level (not in loops, conditions)
  • Only call hooks from functional components or custom hooks
  • Use linter plugin to enforce (eslint-plugin-react-hooks)

Code Examples

Example 1: Counter with useState

import { useState } from 'react';

function CounterComponent() {
  // Declare state variable
  const [count, setCount] = useState(0);

  return (
    <div>
      <h1>Count: {count}</h1>

      <!-- Update state by calling setter -->
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>

      <!-- Or use function form -->
      <button onClick={() => setCount(prev => prev - 1)}>
        Decrement
      </button>

      <button onClick={() => setCount(0)}>
        Reset
      </button>
    </div>
  );
}

export default CounterComponent;

Example 2: useEffect for API Call

import { useState, useEffect } from 'react';

function ProductList() {
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  // Load products when component mounts
  useEffect(() => {
    const loadProducts = async () => {
      try {
        setLoading(true);
        const response = await fetch('http://localhost:5000/api/products');
        const data = await response.json();
        setProducts(data);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    loadProducts();
  }, []); // Empty array = run once on mount

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {products.map(product => (
        <li key={product.id}>
          {product.name} - ${product.price}
        </li>
      ))}
    </ul>
  );
}

export default ProductList;

Example 3: useState with Objects

import { useState } from 'react';

function UserProfile() {
  // State with object
  const [user, setUser] = useState({
    name: 'John',
    email: 'john@example.com',
    age: 30
  });

  // Update single field
  const updateName = (newName) => {
    setUser({
      ...user, // Spread existing data
      name: newName // Override just this field
    });
  };

  // Or use functional form
  const updateEmail = (newEmail) => {
    setUser(prevUser => ({
      ...prevUser,
      email: newEmail
    }));
  };

  return (
    <div>
      <h2>{user.name}</h2>
      <p>Email: {user.email}</p>
      <p>Age: {user.age}</p>

      <input
        value={user.name}
        onChange={e => updateName(e.target.value)}
      />
    </div>
  );
}

export default UserProfile;

Example 4: Cleanup with useEffect (Timer)

import { useState, useEffect } from 'react';

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    // Start timer
    const interval = setInterval(() => {
      setSeconds(s => s + 1);
    }, 1000);

    // Cleanup: clear timer when component unmounts
    return () => {
      clearInterval(interval);
      console.log('Timer cleaned up');
    };
  }, []); // Run once on mount

  return (
    <div>
      <h1>Time Elapsed: {seconds}s</h1>
    </div>
  );
}

export default Timer;

Example 5: Multiple useState & useEffect

import { useState, useEffect } from 'react';

function ProductDetail({ productId }) {
  const [product, setProduct] = useState(null);
  const [quantity, setQuantity] = useState(1);

  // Load product when productId changes
  useEffect(() => {
    fetch(`http://localhost:5000/api/products/${productId}`)
      .then(r => r.json())
      .then(data => setProduct(data));
  }, [productId]); // Re-run if productId changes

  // Log quantity changes
  useEffect(() => {
    console.log('Quantity changed to:', quantity);
  }, [quantity]); // Re-run if quantity changes

  if (!product) return <div>Loading...</div>;

  return (
    <div>
      <h2>{product.name}</h2>
      <p>Price: ${product.price}</p>

      <input
        type="number"
        value={quantity}
        onChange={e => setQuantity(parseInt(e.target.value))}
      />

      <button>
        Add to Cart (Qty: {quantity})
      </button>
    </div>
  );
}

export default ProductDetail;

✅ Best Practices

1. Use Functional Form for State Updates

Why: Ensures you're working with latest state, especially when updates happen quickly

// ❌ Bad - uses stale state
setCount(count + 1);

// ✅ Good - guaranteed latest state
setCount(prevCount => prevCount + 1);

2. Always Spread When Updating Objects/Arrays

Why: React needs new object reference to detect changes

// ❌ Wrong - mutates state
user.name = 'John';
setUser(user);

// ✅ Correct - creates new object
setUser({ ...user, name: 'John' });

3. Don't Call Hooks in Loops or Conditions

// ❌ Wrong - hook order changes
if (condition) {
  useState(...); // Bad!
}

// ✅ Correct - hooks always in same order
useState(...);
useEffect(() => { ... });

4. Clean Up Side Effects

Why: Prevents memory leaks, duplicate listeners, etc.

useEffect(() => {
  const listener = () => console.log('Event');
  window.addEventListener('resize', listener);

  return () => window.removeEventListener('resize', listener);
}, []);

5. Separate Concerns with Multiple useEffect

Why: Easier to understand and maintain

// Load data
useEffect(() => { ... }, [userId]);

// Scroll to top
useEffect(() => { ... }, []);

// Update page title
useEffect(() => { ... }, [title]);

❌ Common Mistakes

1. Forgetting Dependency Array in useEffect

// ❌ Wrong - runs on every render (infinite loop!)
useEffect(() => {
  fetch('/api/data');
});

// ✅ Correct - runs once
useEffect(() => {
  fetch('/api/data');
}, []);

2. Missing Dependencies in Dependency Array

// ❌ Wrong - userId not in dependencies
const [user, setUser] = useState(null);
useEffect(() => {
  fetch(`/api/users/${userId}`);
}, []); // userId used but not listed!

// ✅ Correct
useEffect(() => {
  fetch(`/api/users/${userId}`);
}, [userId]);

3. Setting State in Render

// ❌ Wrong - causes infinite loop
function Component() {
  const [count, setCount] = useState(0);
  setCount(count + 1); // Renders, sets state, renders, sets state...
  return <div>{count}</div>;
}

// ✅ Correct - set state in event handler or useEffect
function Component() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>
    Count: {count}
  </button>;
}

4. Mutating State Directly

// ❌ Wrong - mutates state
items.push(newItem);
setItems(items);

// ✅ Correct - creates new array
setItems([...items, newItem]);

🎓 Interview Q&A

Q1: What's the difference between state and props in React?

Answer:

  • State: Owned by component, can change, managed with useState
  • Props: Passed from parent, read-only, passed as function arguments
  • Rule: State for data that changes, props for passing data down

Q2: What does the dependency array in useEffect do?

Answer: Controls when useEffect runs:

  • No array: Runs after EVERY render (⚠️ can cause infinite loops)
  • []: Runs ONCE on mount - use for initialization
  • [dep1, dep2]: Runs when any dependency changes
  • Lesson: If you use a variable in useEffect, it should be in dependencies

Q3: How do you prevent infinite loops with useState and useEffect?

Answer: Common cause is setting state inside useEffect without dependencies:

// ❌ Infinite loop
useEffect(() => {
  setCount(count + 1); // Sets state, triggers re-render, effect runs again
});

// ✅ Fixed with empty dependency array
useEffect(() => {
  console.log('Runs once');
}, []);

// ✅ Or include count in dependencies (still reruns though)
useEffect(() => {
  setCount(prev => prev + 1);
}, [count]); // Explicit dependency
